# Bash #

## Простая работа с файловой системой ##

1. Перемещение файлов
    ``` bash
    $ mv [опции] источник назначение
    Опции:
    -i  спрашивать при перезаписи существующего файла
    -n  не перезаписывать существующее без спроса
    -u  перезаписать только более новыми файлами
    -v  подробный вывод, что сейчас переносится
    -b  сделать бекап перезаписываемого файла (добавляет ~ или суффикс через --suffix)
    ```
    Типичные сценарии:  
    1.1. Простое перемещение файла  
    ``` bash
    $ mv old/path/file.smthing new/path/
    ```
    1.2. Переименование файла  
    ``` bash
    $ mv oldname.file newname.file
    ```
    1.3. Перемещение множества файлов через подстановки  
    ``` bash
    $ mv *.log /path/to/move/ # переместит все файлы из текущей директории, заканчивающиеся на .log
    $ mv /path/to/smthing/*.log /path/to/move # переместит все файлы из директории /path/to/something/ заканчивающиеся на .log
    ```

2. Копирование файлов
    ``` bash
    $ cp [опции] источник назначение
    Опции:
    -i  спрашивать при перезаписи файла
    -n  не перезаписывать существующие файлы
    -u  перезаписывать только более новые файлы
    -v  показать прогресс копирования
    -f  принудительная перезапись файлов
    -r  рекурсивно произвести копирование всего источника (директории)
    -a  архивный режим копирования сохраняется всё, как в источнике (права, ссылки, временные метки и т.п.)
    -p  сохранить права и атрибуты источника
    -L  копировать содержимое ссылок, а не ссылки
    -l  создаёт новые жесткие ссылки, вместо полного копирования
    -s  создаёт символические ссылки, вместо копирования
    --backup[=SUFFIX]   создание резервной копии перезаписываемых файлов с суффиксом
    ```
    2.1 Простое копирование  
    ``` bash
    $ cp name.file /new/path/to/copy/
    ```
    2.2 Копирование каталога со всем его содержимым
    ``` bash
    $ cp -r /path/directory /copy/path
    ```
    2.3 Архивное копирование без потери атрибутов файлов
    ``` bash
    $ cp -a /path/directory /copy/path # Обычно для резервного копирования перед изменениями
    ```
    2.4 Копирование всего содержимого директории без самой директории
    ``` bash
    $ cp -r /source/. /destination/ # символ . говорит "скопируй всё"
    ```
3. Простая работа с файлами
   
    3.1. Простое создание файлов
    ``` bash
    $ touch /путь/имя
    ```
    Просто создаёт файл по указанному пути, при применении к существующему файлу - обновляет его временную метку модификации
    ``` bash
    $ > name.file
    ```
    Создание пустого файла или очистка содержимого файла с помощью перенаправления вывода. Пустой вывод = пустой файл.

    3.2. Создание файлов с содержимым

    Для создания файлов с содержимым обычно производится перенаправление вывода от любых команд, которые выводят что-либо в потоки stdout, stderr, stdwrn и делается это ключевым символом >, который как бы указывает откуда куда перенаправить вывод.
    ``` bash
    $ cat file.txt > newfile.txt # вывод всего file.txt в новый файл
    $ echo "helloworld!" > helloworld.txt # вывод печати echo в файл
    $ printf "My name,\nAlice!" > Alice.name.txt # вывод форматированного текста (с переносом строки) в файл
    ```

    3.3. Сложное создание файлов

    Для заполнения многострочных файлов используются ```EOF``` последовательности в купе с символом перенаправления вывода ```>```
    ``` bash
    cat > config.conf << EOF
    server {
        listen 80;
        server_name example.com;
    }
    EOF # Всё, что заключено между EOF отправляется на ввод cat как файл, а вывод cat отправляется в config.conf с перезаписью
    ```
    Чтобы не перезаписывать файл, а доплнить, можно использовать ```>>```
    ``` bash
    cat >> config.conf << EOF
    server {
        listen 80;
        server_name example.com;
    }
    EOF # Всё, что заключено между EOF отправляется на ввод cat как файл, а вывод cat отправляется в config.conf с дополнением
    ```
    Команда tee так же позволяет сложным способом взаимодействовать с содержимым файла. Данная команда через stdin получает на вход данные из, например, других команд или интерактивно через терминал пользователя и записывает полученное в файл(ы) указанные в параметрах, а также одновременно выводит на экран полученное.

    ``` bash
    $ tee [опции] файл [файл...]
    -a  добавить в файл, а не перезаписывать
    ```
    Основные сценарии использования:
    ```bash
    $ tee file.text # запишет всё, что вы будете вводить в консоли, пока не получит сигнал прерывания CTRL+C или иной
    $ cat service.log | tee -a text.file # дополнит содержимым файла service.log содержимое файла text.file. Символ | перенаправляет вывод одной команды (поток STDOUT в поток STDIN другой команды)
    ```
    Так же с tee используется форматирование heredoc:
    ```bash
    $ cat << 'EOF' | tee файл1 файл2 ... >/dev/null
    многострочный
    текст
    EOF
    ```

    3.4. Удаление файла

    ``` bash
    $ rm /path/name.file # rm без параметров не удаляет директории, только файлы. 
    ```
4. Работа с директориями
    4.1. Создание директории
    ``` bash
    $ mkdir [опции] путь
    -p  создаёт недостающие директории по указанному пути
    -m  позволяет сразу настроить права доступа создаваемой директории (-m 775 например)
    -v  выводить сообщение о каждой созданной директории
    ```
    4.2. Создание временных директорий и файлов
    ``` bash
    $ mktemp -d # создаст и вернёт путь к созданной временной директории
    $ mktemp # создаст и вернёт путь к созданному временному файлу
    ```
    4.3. Удаление директорий
    ```bash
    # вариант 1
    $ rmdir /path/to/dir # удаляет только пустой каталог
    # вариант 2
    $ rm -r /path/to/dir # удаляет каталог и всё его содержимое
    # Вариант 3 (без подтверждения удаления)
    $ rm -rf /path/to/dir # так же, как вариант 2, но не спросит подтверждений
    ```
    4.4. Перемещение и переименование директорий

    Работает стандартная команда mv
    ```bash
    $ mv oldname newname
    $ mv /old/path/directory /new/path/
    ```
5. Ориентирование в файловой системе

    У каждой сессии терминала есть рабочая директория - директория, в которой выполняются команды, в параметрах которых не указан полный путь до файлов с которыми работать. 

    текущая рабочая директория чаще всего отображается левее символа-приглашения командной строки, например: 

    ```bash
    user@host:/opt/media $ # /opt/media - текущая рабочая директория
    ```
    
    Также текущую рабочую директорию можно получить командой pwd:
    ```bash
    $ pwd
    /var/log # вывод /var/log означает, что команда pwd была выполнена, когда рабочей директорией была /var/log
    ```

    5.1. перемещение

    Для смены рабочей директории используется команда ```cd``` и некоторые сокращения, чтобы не набирать полный путь каждый раз: ```.``` - точка означает текущая директория,  ```..``` - две точки - родительская директория, ```~``` - тильда ведёт в домашнюю директорию.

    ``` bash
    $ cd ../ # переместится в родительскую директорию текущей директории
    $ cd ./directory # переместится из текущей директории в поддиректорию directory
    $ cd ../directory # переместится в поддиректорию directory родительской директории 
    $ cd /var/log # переместится в директорию /var/log
    $ cd ~ # переместится в домашнюю директорию пользователя (обычно /home/{username}/)
    ```

    5.1.1. Стековый подход к работе с директориями

    Для стекового взаимодействия с директориями используются команды ```pushd``` и ```popd```, а также ```dirs```
    
    ``` bash
    $ pushd /var/log
    /var/log $ # тут выполняем какие-то команды и надо переместится в другой каталог, допустим /etc/bin
    /var/log $ pushd /etc/bin
    /etc/bin $ # тут тоже, что-то делаем и надо вернуться в исходный каталог
    /etc/bin $ dirs
    /etc/bin
    /var/log
    /home/{username}
    /etc/bin $ popd
    /var/log $ popd
    $ # вернулись в самый исходный каталог
    ```

    5.2. Просмотр содержимого директории

    Для просмотра содержимого директории применяется команда ls с различными параметрами, чаще всего это параметры ```-l```,```-a```,```-h```;

    ``` bash
    $ ls [опции] [путь]
    -l  вывод подробной информации о содержимом директории (права, владелец, группа, дата изменения)
    -a  вывод в т.ч. скрытых файлов и папок
    -h  человеко-читаемые размеры файлов (мб, кб, гб)
    -R  рекурсивный вывод подкаталогов (чаще удобнее пользоваться командой tree, доступной для установки через apt в большинстве дистрибутивов)
    -t  сортировка по дате изменения
    -S  сортировка по размеру
    -r  обратный порядок сортировки
    -d  показывать информацию о директории, а не о содержимом
    --color подсветка директорий (включена, обычно, по умолчанию)
    ```

    Другая полезная команда, которая часто не установлена, но доступна для установки через менеджеры пакетов это ```tree```

    ```bash
    $ tree [опции] [путь]
    -d  показать только дерево директорий
    -L N    показать только на N уровней вглубь
    -a  показать так же скрытые директории
    -h  человеко-читаемые размеры файлов (вместе с --du)
    ```
    
    5.3. Определение типа файла

    Часто в скриптах может потребоваться определить тип файла, с которым взаимодействует скрипт, в таком случае необходимо воспользоваться утилитой ```file``` или ```stat```

    ``` bash
    $ file /etc/passwd  # ASCII text
    $ file /bin/ls      # ELF 64-bit executable
    $ file image.png    # PNG image data
    $ stat /etc/hosts   # Выведет размер, inode, права и временные метки
    ```

    5.4. Поиск файлов

    Самая частая задача системного администратора - поиск файлов и директорий, для определения местоположения конфигурационных файлов, файлов баз данных и др.

    Для этих целей применяется 2 очень мощных инструмента командной строки: ```find``` и ```grep```

    ``` bash
    $ find [путь] [параметры]
    Параметры(выражения для поиска):
    -name "шаблон"  # поиск с учётом регистра по простому шаблону (-name "*.log" - все файлы с расширением  .log)
    -iname "шаблон" # поиск без учёта регистра по простому шаблону
    -type [тип]     # поиск по типу: d - директория, f - файл
    -size [размер]  # поиск файлов по размеру: +N[k,m,g] - больше указанного размера, -N[k,m,g] - меньше указанного размера
    -mtime [дни]    # поиск файлов, модифицированных в определённое время: +N - изменён более N дней назад, -N - изменён менее N дней назад. 
    -mmin [минуты]  # так же, как mtime, только в минутах.
    -user "username"    # поиск файлов с конкретным владельцем
    -group "group"  # поиск файлов с конкретной группой
    -empty  # поиск пустых файлов и директорий
    -path "шаблон"  # поиск таких файлов, путь к которым соответствует шаблону например -path "*enabled*" будет соответствовать для включенных конфигураций nginx
    -regex "шаблон" # поиск таких файлов, полный путь которых соответствует регулярному выражению
    -perm NNN   # поиск файлов с указанными правами доступа (например -perm 777 -delete удалит все файлы с правами 777)
    -ls # вывести найденные файлы аналогично команде  ls -l
    -delete # удалить найденные файлы
    -exec команд {} \; # для каждого файла выполнить указанную команду (вместо {} подставляется имя каждого файла последовательно)
    -exec команда {} + # для всех файлов разом выполнить команду (вместо {} подставляются все имена файлов разом)
    -ok команда {} [\;,+] # то же, что exec, но перед выполнением команды запросит подтверждение
    ```

    Так же менее мощной, но в то же время простой и быстрой утилитой является ```locate``` для которой необходимо существование базы данных с информацией о структуре директорий и файлов, в них содержащихся, которая обновляется командой ```updatedb``` с правами администратора. 

    ``` bash
    $ locate [опици] путь
    Опции:
    -i  # поиск без учёта регистра
    -r "регулярное выражение"   # поиск файлов, удовлетворяющих регулярному выражению
    -l N    # ограничить вывод N строками
    # updatedb # обновить базу-данных (чаще всего делается через cron задачу)
    ```

    5.5. Полнотекстовый поиск

    Команда ```grep``` - инструмент, вызываемый даже чаще, чем ```cd```. Позволяет осуществлять полнотекстовый поиск по содержимому файлов, а также фильтровать стандартный ввод.

    ``` bash
    $ grep [опции] шаблон [файлы/директории]
    Опции:
    -i  # игнорировать регистр
    -r или -R   # рекурсивный поиск по директориям
    -n  # вывод с номерами строк
    -l  # вывод только имён файлов, где есть совпадение 
    -L  # вывод только имён файлов, где НЕТ совпадения
    -v  # показывать строки, НЕ соответствующие шаблону
    -w  # искать соответствие шаблону как целое слово, а не его часть
    -c  # подсчёт количества совпадений
    -A N    # показать N строк после совпадения
    -B N    # показать N строк до совпадения
    -C N    # показать N строк вокруг совпадения
    -e шаблон    # использовать несколько шаблонов (-e pattern1 -e pattern2)
    -H  # всегда показывать имя файла
    -h  # не показывать имя файла
    ```

    5.6. Примеры:

    ``` bash
    # Поиск файлов, содержащих выражение в директории и поддиректориях
    $ grep -r "error" /var/log
    # поиск в переданном файле выражения с выводом номера строки
    $ grep -n "error" /var/log/nginx/nginx.log
    $ cat /etc/nginx/nginx.conf | grep -n "listen" 
    # поиск файлов, изменённых менее дня назад с поиском в них ошибок:
    $ find /var/log -name "*.log" -mtime -1 -exec grep -Hn "error" {} \;
    # поиск сразу во всех файлах
    $ find /var/log -name "*.log" -exec grep -ln "error" {} +
    ```

    5.7. Современная альтернатива ripgrep (rg)
    
    Не предустанавливается поумолчанию, требует установки и обычно доступна под именем пакета ```ripgrep``` (```apt install ripgrep``` или ```cargo install ripgrep```)

    ``` bash
    $ rg [опции] шаблон [файлы/директории]
    Опции:
    1. Вывод и форматирование:
        -l  # Вывод только имён файлов, где обнаружено совпадение
        -L  # Вывод только имён файлов, где совпадения не обнаружено
        -с  # Вывод подсчёта совпадений
        -n  # Номера строк, где обнаружено совпадение
        -o  # Вывод только совпавшей части строки
        -A N, -B N, -C N    # Вывод N строк: До, После, Вокруг совпадения
        --stats # Статистика поиска
        --pretty    # Красивый вывод (цвета, заголовки, номера строк)
    2. Фильтрация
        -t тип  # поиск в файлах конкретного типа (напр. -t py поиск только файлов с раширением .py)
        -T тип  # исключить из поиска тип данных
        --hidden    # поиск по скрытым файлам и директориям
        --no-ignore # не игнорировать .gitignore
        -z  # поиск в архивах
        -g  # включить файлы по glob (т.е. поиск файлов по шаблону сопоставления со спецсимволами)
    3. Поиск
        -i  # игнорировать регистр
        -s  # не ингнорировать регистр
        -S  # если шаблон в нижнем регистре - игнорирует регистр, иначе - учёт
        -F  # шаблон воспринимается как литерал, а не регулярное выражение и ищется вхождение литерала
        -U  # многострочный поиск
        -v  # инверсия шаблона (т.е. не встречается указанный шаблон)
        -x  # только если строка полностью соответствует шаблону
        -r  # заменить совпадения при выводе
    4. Управление
        -j N    # поиск в N потоков
        --sort  # сортировка результатов
        --files # Вывести список обработанных файлов
        --type-list # вывести известный список типов файлов (для опции -t)
        --debug # отладка (почему файлы игнорируются)
    ```

    Примеры:

    ```bash
    # Базовый поиск
    $ rg "TODO" # рекурсивный поиск в текущей директории
    $ rg -n "error" /var/log # рекурсивно найти ошибки в логах
    $ rg -i "user" ./ # рекурсивно найти без учёта регистра вхождение user  
    # Поиск только имён файлов
    $ rg -l "password" ./project # поиск в директории project имён файлов, в которых есть вхождение password
    # Поиск контекста и статистики
    $ rg -C 5 "exception" app.log  # 5 строк вокруг
    $ rg --stats "TODO"            # статистика по завершении
    # Продвинутый поиск
    $ rg "^func\s+\w+\(\)" # строки, начиающиеся с func name()
    # Перебор файлов
    $ rg --files -g "*.rs"  # вывести файлы с расширениме .rs
    ```

    Управляющие GLOB конструкции:
    1. ```*``` - любая последовательность символов (rg --files -g "*.rs" - my.rs, you.rs, name.rs)
    2. ```?``` - любой 1 символ. (file?.log - file1.log, file2.log, fileA.log etc.)
    3. ```[abc]``` - любой 1 символ из указанных (file[abc].log - filea.log, fileb.log, filec.log)
    4. ```**``` - любая вложенность директорий (найдёт все вхождения шаблона на любой глубине от текущей директории)

6. Просмотр содержимого файлов
   
    6.1. ```cat``` - простой вывод текстового содержимого в консоль

    ``` bash
    $ cat [опции] файлы
    Опции:
    -n  # нумерация строк
    -b  # не нумеровать пустые строки
    -s  # несколько пустых строк сжимаются в одну
    -Е  # в конце строки выводить $
    -T  # вместо табуляции выводить символ ^I
    -A  # печатать непечатные символы (эквивалент  -vET)
    # сценарии использования
    # Вывести содержимое файла
    $ cat file.txt

    # Объединить несколько файлов
    $ cat file1.txt file2.txt > combined.txt

    # Добавить один файл в конец другого
    $ cat append.txt >> target.txt

    # Просмотреть файл с нумерацией строк
    $ cat -n config.conf

    # Быстрое создание файла (с завершением по Ctrl+D)
    $ cat > newfile.txt
    ```

    6.2. ```head``` - вывод первых строк файла ```tail``` - вывод последних строк файла

    По-умолчанию обе команды выводят 10 строк, количество выводимых строк определяется параметром ```-n``` т.е.  ```head -n 1000 file.txt``` выведет первую 1000 строк файла, ```tail``` выведет последнюю тысячу. 

    6.3. ```less``` - многостраничный просмотр, с возможностями поиска, прокрутки экранов и перехода по строкам

    ``` bash
    $ less [опции] [файл]
    Опции:
    -N  # нумеровать строки
    -S  # обрезать длинные строки, а не переносить
    -F  # выход из less если содержимое умещается на 1 экране
    -R  # показать цветной вывод упралвяющих последовательностей (переносы, табуляция, возврат каретки и т.п.)
    -X  # не очищать экран при выходе
    -i  # искать без учёта регистра если шаблон в нижнем регистре
    -I  # искать полностью без учёта регистра
    -p "шаблон" # поиск по шаблону
    Горячие клавиши внутри less:
    пробел, f - перелистнуть 1 страницу вперёд
    b - 1 страница назад
    j, ↓, e - одна строка вперёд
    k, ↑, y - одна строка назад
    g - в начало файла
    G - в конец файла
    N G - перейти на строку N
    /pattern - поиск по паттерну вперёд
    ?pattern - поиск по паттерну назад
    &pattern - показать только строки совпадающие с паттерном
        n - следующее совпадение
        N - предыдущее совпадение
    = - информация о позиции (строки, процент файла)
    h - справка
    q - выйти
    # полезный трюк с подсветкой:
    $ tail -f app.log | grep --color=always "ERROR" | less -R
    ```

7. Структура каталогов файловой системы

    7.1. ```/``` - корневой раздел, является точкой средоточения всей файловой системы. Все директории имеют начало - корневой раздел.

    7.2. ```/bin``` - основные пользовательские команды (исполняемые файлы) здесь находятся ls, cp, mv, cat, echo, bash и др. команды

    7.3. ```/sbin``` - системные команды администратора. Доступны только администратору. fdisk, mkfs, mount, iptables, systemctl, reboot и др.

    7.4. ```/etc``` - системные файлы конфигурации. Конфигурация системных утилит и установленных глобально программ, вроде nginx, kaddy, mysql, mariadb и др.

   Сетевые настройки расположены: ```/etc/network/```, файл хостов: ```/etc/hosts```, файл dns: ```/etc/resolv.conf```

   Конфигурации служб: ```/etc/nginx/```, ```/etc/apache2/```, ```/etc/sshd/```, ```/etc/apt/```

   Системные файлы: ```/etc/fstab``` - монтирование дисковых разделов, ```/etc/passwd``` - файл о пользователях системы, ```/etc/shadow``` - файл с хешами пользовательских паролей.

   Так же тут располагаются файлы конфигурации инициализации систем: ```/etc/crontab```, и каталоги ```/etc/init.d/```, ```/etc/systemd/``` и другие, в зависимости от дистрибутива.
   
    7.5. ```/home/``` - директория для пользовательских каталогов. Директория пользователя user будет находится в ```/home/user/```

    7.6. ```/root/``` - домашняя директория суперпользователя (администратора)

    7.7. ```/tmp``` - директория временных файлов и каталогов, очищающаяся при перезагрузке. Все пользователи имеют доступ, но удалять могут только созданное ими самими.

    7.8. ```/usr/``` - директория для пользовательских программ и данных. 
        
    ```/usr/bin``` - большинство исполняемых команд  
    ```/usr/sbin``` - дополнительные административные инструменты  
    ```/usr/lib/``` - подключаемые библиотеки для программ из ```/usr/bin/``` и ```/usr/sbin```  
    ```/usr/local``` - программы собранные из исходников в системе, каждая содержит свой ```bin/```, ```lib/```, ```etc/```.  
    ```/usr/share``` - файлы данных: иконки, шрифты, изображения, man-страницы и т.п.  
    ```/usr/include``` - заголовочные файлы для разработки на C/C++  

    7.9. ```/opt/``` - стороннее ПО и проекты, не следующие структуре usr директории. Обычно имеют свои структуры директорий. Так же сюда устанавливаются большие по объёму программы. И администраторы держат здесь свои docker программы.

    7.10. ```/boot/``` - файлы загрузчика
        
    Ядро: ```vmlinuz-*```  
    Образы: ```initrd.img-*```  
    Конфигурация загрузчика: ```grub/``` (или иного) 

    7.11. ```/dev``` - файлы блочных устройств

    Диски и разделы: ```/dev/sda```,```/dev/sda1```, ```/dev/nvme0n1```   
    Терминалы: ```/dev/tty```,```/dev/pts```  
    Нулевое устройство (ввод-вывод нули) и устройство случайных данных: ```/dev/null``` и ```/dev/random```  или ```/dev/urandom```  

    7.12. ```/proc``` - виртуальная файловая система процессов, запущенных в системе

    ```/proc/cpuinfo```   - информация о процессоре   
    ```/proc/meminfo```   - информация о памяти  
    ```/proc/loadavg```   - средняя нагрузка на систему  
    ```/proc/filesystems```   - поддерживаемые ФС  
    ```/proc/{PID}```     - информация о процессе с указанным PID.  

    7.13. ```/sys``` - появившаяся в системах с ядром 2.6 и выше дииректория, позволяющая управлять устройствами и модулями ядра

    7.14. /run - временные файлы со сведениями о выполняемых процессах

    ```/run/process.pid``` - PID идентификатор процесса  
    ```/run/user/1000/``` - информация о сокетах, разделяемой памяти, точках монтирования  
    ```/var/run``` - теперь является символической ссылкой на ```/run```  

    7.15. ```/mnt/``` - директория, куда следует монтировать подключаемые устройства

    7.16. ```/media``` - директория для автоматического монтирования подключаемых устройств (напр. флешки)

    7.17. ```/srv/``` - директория разделяемых ресурсов.  

    ```/srv/ftp``` - директория FTP-репозитория
    ```/srv/www``` - директория для web-сервера и т.п.

    7.18. Дополнительно:

    ```lost+found``` - что удалось автоматически восстановить в случае сбоя ФС  
    ```/swapfile``` - файл подкачки оперативной памяти  
    ```/lib``` - библиотеки для ```/bin```  

    7.19. Общий вид ФС:

    ``` bash
    /
    ├── bin/ -> usr/bin
    ├── sbin/ -> usr/sbin
    ├── etc/
    ├── home/
    │   └── user/
    ├── root/
    ├── var/
    │   ├── log/
    │   ├── lib/
    │   └── tmp/
    ├── usr/
    │   ├── bin/
    │   ├── lib/
    │   ├── local/
    │   └── share/
    ├── boot/
    ├── dev/
    ├── proc/
    ├── sys/
    ├── tmp/
    ├── run/
    ├── opt/
    ├── mnt/
    ├── media/
    └── srv/
    ```

    Почему это важно знать администратору:

    - Понимание, где искать логи – ```/var/log```

    - Где менять конфигурации – ```/etc```

    - Куда устанавливать своё ПО – ```/usr/local``` или ```/opt```

    - Почему ```df -h``` показывает разные разделы – ```/home```, ```/var```, ```/boot``` могут быть на отдельных разделах.

    - Безопасность – правильные права на каталоги (```/tmp``` со ```sticky bit```, ```/home``` с приватными правами).

    - Диагностика – через ```/proc``` и ```/sys``` получать информацию о системе без дополнительных утилит
