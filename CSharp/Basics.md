## Первые шаги

### Типы данных
- Примитивы:
    | Тип данных    | Размерность                   | Описание |
    | ----------    | -----------                   | -------- |
    | `sbyte`       | [-128;+127]                   | знаковое 8-ми битное целое |
    | `byte`        |  [0;255]                      | беззнаковое 8-ми битное целое |
    | `short`       |  [-32768;+32767]              | знаковое 16-битное целое |
    | `ushort`      |  [0;65535]                    | беззнаковое 16-битное целое |
    | `int`         |  [-2млрд;+2млрд]              | знаковое 32-битное целое |
    | `uint`        |  [0;4млрд]                    | беззнаковое 32-битное целое  |
    | `long`        |  [-9*10^18; +9*10^18]         | знаковое 64-битное целое |
    | `ulong`       |  [0;18*10^18]                 | беззнаковое 64-битное целое |
    | `float`       |  [+-1.5*10^-45;+-3.4*10^38]   | 32-битное число с плавающей точкой (точность 7 знаков) |
    | `double`      |  [+-5*10^-324;+-1.7*10^308]   | 64-битное число с плавающей точкой (точность 15 знаков) |
    |  `decimal`    |  [+-1*10^-28;+-7.9*10^28]     | 16-байтное число с плавающей точкой (точность 29 знаков) |  
    |  `char`       | UTF-16                        | 2-байтовое символьное значение в кодировке UTF-16 |
    |  `bool`       | [true/false]                  | 1-байтовое логическое |

    К особым типам данных можно отнести `string` и `object`. Первый, несмотря на объявление типичное для примитивов, является ссылкой на объект (экземпляр класса System.String), так же надо помнить, что данный тип `иммутабелен`, что значит, что изменение всегда приводит к *созданию* нового экземпляра. Второй является корневым объектом ООП-иерархии в C#. Все классы по умолчанию наследуют тип Sytem.Object, посему могут быть приведены к нему. Можно назвать это аналогом `void*` из C++, хотя таковым не является. 
    
    Практические рекомендации:
    - Используй `byte` для работы с массивами данных, так удобнее их передавать в потоки, например в вебсокеты. 
    - Циклы итерируй через `int`, потому что внутри оно оптимизировано для работы с `int`
    - Флаги состояния используй любой `целочисленный тип`, в зависимости от размера ты сможешь использовать до 64 различных флагов состояний. Обычно, столько не нужно и хватает `byte`. 
    - Для ID сущностей используй подходящую размерность. Если сущностей гарантировано меньше 65тыс, то тебе подойдёт `ushort`. В противном случае хватает чаще всего `uint`
    - Не забывай про безопасность памяти и используй `Nullable`: `int? arg = Nullable<int>` оверхед по памяти 1 байт, но безопасно.
- Nullable-типы (`int?`, `Nullable<T>`)
    Nullable-типы это специальная обёртка, добавляющая к базовому (или не очень) типу данных байт данных, где содержится специальный флаг, определяющий пуст объект или нет. 
    
    ```csharp
    int? age = null;              // краткая форма
    Nullable<int> age2 = null;    // полная форма — синоним
    ```
    
    Используются nullable типы в тех случаях, когда значение по-умолчанию для объекта не определено алгоритмом. Условно если мы итерируемся по массиву, то индекс, который мы ищем может быть nullable типом, и значение null после поиска укажет нам, что ничего найдено не было:

    ```csharp
    uint arr = [7, 15768, 45518, 2798145667];
    ushort? srch = null;
    for(uint i=0; i<arr.length; i++){
        if(arr[i]=70){
            srch=i;
        }
    }
    if(srch==null) Console.WriteLine("Not found");
    ```

    Nullable объекты имеют так же некоторое количество синтаксического сахара упрощающего некоторые взаимодействия с ними:
    - Оператор `??` позволяет задать значение исходя из того null объект или нет
        ```csharp
        int result = nullableInt ?? 0; // если null, то будет 0
        ```
    - Оператор `??=` позволяет присвоить некоторое значение, если до этого оно было `null`
        ```csharp
        nullableInt ??= 0; // если nullableInt был null, то назначить ему 0
        ```
    Nullable лучше всего использовать, когда приложение общается с внешними источниками данных (базы данных например), откуда может прийти null и не гарантируется, что null не будет передан. 

- `var` и вывод типов
    Проблема: не всегда очевидно, какой тип данных получится после выполнения некоторого выражения и на этапе разработки может понадобиться вывести тип данных из выражения, результат которого присваивается переменной. Определение типа осуществляется на этапе компиляции и не является динамическим.

    ```csharp
    var name = "John";           // string
    var items = new List<int>(); // List<int>
    var dict = new Dictionary<string, List<int>>(); // сложный тип
    ```
