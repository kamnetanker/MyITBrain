# 🧩 Практический проект: Универсальная панель управления сервисами (Service Manager)

> **Цель:** Создать систему, которая позволяет управлять несколькими сервисами через единую веб-панель, без необходимости писать код для каждого нового сервиса. Всё, что нужно для интеграции — JSON-манифест, описывающий API сервиса.

---

## 📌 Идея

Мы строим **Service Manager** — приложение, которое:
1. Хранит **реестр сервисов** (Service Registry) и их JSON-манифесты.
2. Предоставляет **веб-панель**, которая динамически генерирует UI на основе манифеста.
3. Позволяет **вызывать методы сервисов** через панель (прокси-запросы).
4. Умеет **регистрировать новые сервисы** без перезапуска (через API или файлы).

---

## 🧱 Компоненты системы

| Компонент | Назначение |
|-----------|------------|
| **Service Registry** | Хранит список сервисов и их манифестов. Умеет читать из папки `./Manifests`, принимать по HTTP, хранить в памяти или БД. |
| **Manifest Parser** | Парсит JSON-манифест, валидирует его схему, преобразует во внутренние объекты C#. |
| **Dynamic UI Generator** | На основе манифеста генерирует HTML-формы (или Blazor-компоненты) для каждого метода. |
| **API Proxy** | Принимает запросы от панели, формирует HTTP-запрос к целевому сервису, возвращает ответ. |
| **Service Scanner** | (Опционально) Фоновый сервис, который сканирует папку `./Manifests` и обновляет реестр при появлении новых файлов. |

      ]
    }
  ],
  "schemas": {
    "UserDto": {
      "type": "object",
      "properties": {
        "id": { "type": "integer" },
