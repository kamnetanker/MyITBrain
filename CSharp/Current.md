## Планируемые практические темы

### Создание приложения с поддержкой плагинов

**Цель:** научиться проектировать приложения так, чтобы третьи лица могли расширять их функциональность без изменения основного кода (десктоп, мобилка, игры).

**Подтемы:**
- Архитектура "Хост → Контракт → Плагин"
- Создание общего контракта (интерфейсы `IPlugin`, `ICommand` и т.п.)
- Динамическая загрузка плагинов через рефлексию
- Современная изоляция: `AssemblyLoadContext` и `AssemblyDependencyResolver`
- Интеграция с Dependency Injection (DI)
- Обнаружение плагинов через кастомные атрибуты
- Жизненный цикл плагина (загрузка → инициализация → активация → выгрузка)
- Ограничения мобильных платформ (iOS AOT, Android)
- Безопасность и песочница

---

**Практический пример:** создание калькулятора с командами, где каждая команда — отдельный плагин (пример из документации Microsoft).

### Разработка игры на Godot / Redot Engine (C#)

**Цель:** освоить создание игр на Godot (или его форке Redot) с использованием C# — от настройки среды до публикации на десктоп и мобильные платформы.

**Подтемы:**
- Установка Godot (.NET-версия) и настройка редактора под C#
- Создание первого скрипта на C# (вместо GDScript)[reference:0]
- Основы Godot API из C#: Node, SceneTree, сигналы, таймеры[reference:1]
- Работа с 2D/3D-физикой и анимациями через C#
- Интеграция сторонних .NET-библиотек (NuGet) в игровой проект
- Экспорт проекта на десктоп (Windows, Linux, macOS)[reference:2]
- Экспорт на Android и iOS (AOT-компиляция, особенности)[reference:3][reference:4]
- Создание игровых систем: состояния, компоненты, инвентарь, сохранения

**Практический пример:** создание простой 2D-игры (арканоид/платформер) с нуля на C#.

**Ресурсы:** Microsoft Godot C# Essentials[reference:5], официальная документация Godot C#[reference:6].

---

### Мобильное приложение-чат для локальной сети (P2P)

**Цель:** разработать кроссплатформенное мобильное приложение для обмена сообщениями напрямую между устройствами в одной локальной сети, без центрального сервера.

**Подтемы:**
- **Обнаружение соседей (Discovery):** mDNS / Zeroconf / Bonjour[reference:7] — автоматический поиск других устройств в локальной сети.
- **Peer-to-Peer соединение:** установка прямого TCP-соединения между найденными устройствами[reference:8].
- **Альтернативный подход:** WebRTC с DataChannel для P2P-связи (обход NAT, работа через интернет)[reference:9][reference:10].
- **Обмен сообщениями:** текстовые сообщения, уведомления о статусе (онлайн/офлайн), история чата.
- **Кроссплатформенность:** .NET MAUI / Avalonia UI / Uno Platform для единого кода под Android и iOS.
- **Разрешения:** сетевые разрешения на мобильных платформах (Android, iOS)[reference:11].
- **Безопасность:** шифрование сообщений (опционально).

**Практический пример:** консольный или мобильный чат-клиент, который автоматически находит соседей в Wi-Fi сети и позволяет обмениваться сообщениями.

**Ресурсы:** Zeroconf (NuGet)[reference:12], примеры P2P-чатов на GitHub[reference:13][reference:14].

### Веб-приложение "Менеджер задач" (Todo API) на ASP.NET Core

**Цель:** создать полноценный бэкенд для управления задачами с хранением в базе данных, авторизацией, логированием и интеграцией с внешним API. Это классический проект, который показывает, как работают вместе все ключевые Middle-технологии.

**Подтемы:**
- Проектирование моделей данных (Entity Framework Core, Code First, миграции)
- RESTful API на ASP.NET Core (контроллеры, маршрутизация, модель-биндинг, валидация)
- Dependency Injection (регистрация сервисов, работа с ILogger, IConfiguration)
- Работа с БД: PostgreSQL / SQLite / SQL Server через EF Core (запросы, LINQ, отношения)
- Аутентификация и авторизация (JWT, регистрация/вход пользователей)
- Кэширование (IMemoryCache, IDistributedCache + Redis)
- Логирование (Serilog / ILogger, structured logging, запись в файл или консоль)
- Фоновые задачи (IHostedService / BackgroundService) — например, очистка старых задач
- Интеграция с внешним API (например, получение курсов валют или погоды через HttpClient)
- Тестирование: Unit-тесты (xUnit + Moq), интеграционные тесты (TestContainers)
- Документирование API (Swagger / OpenAPI)

**Практический пример:** API для управления задачами с пользователями, категориями, приоритетами, сроками выполнения и уведомлениями.

**Ресурсы:** официальные туториалы Microsoft по ASP.NET Core, EF Core, JWT, xUnit, Serilog.
