# Полиморфизм

Полиморфизм - финальный кирпич в фундаменте ООП, наравне с инкапсуляцией и наследованием задающий принципы программирования в ООП стиле.

Полиморфизм - это способность иметь множество реализаций при единообразии интерфейса взаимодействия.

При `полиморфизме` объекты `производного` класса могут приводиться к `базовому` таким образом, что **тип с которым работает** алгоритм не соответствует **реальному типу объекта**, а является `первородным` для реального типа или родительским. `virtual-методы` `первородного` класса могут переопределять поведение объектов `дочерних` классов (`override`) и **определение** точной `реализации` для выполнения происходит на лету во время `выполнения`.

Три основных вида полиморфизма:

- `Полиморфизм подтипов` (Runtime-полиморфизм) - реализуется через механизм `наследования` и `переопределения` поведения `базового` класса через `virtual`/`override` методы или `интерфейсы`. Разрешение адресов вызова происходит в `runtime` среде.
- `Параметрический полиморфизм` (`Generics`) - реализуется через универсальные `типы` и `методы`. **Выбор** реализации определяется на этапе `компиляции`.
- `Перегрузка методов` (Compile-time-полиморфизм) - реализуется через совокупность эквивалентных `сигнатур` методов, различных лишь `параметрами` и `возвращаемыми` типами. **Разрешение** реализации происходит на этапе `компиляции`.

Если определение вызова на этапе компиляции очевидно - компилятор выводит из выражения тип результата и на основании выведенного типа формирует байт-код для места, где используется перегрузка или параметрический полиморфизм, то определение на этапе выполнения требует дополнительного пояснения.

Механика работы определения на этапе выполнения заключается в том, что для `virtual`/`override` конструкций создаётся таблица виртуальных методов и при достижении выполнения место вызова некоторого метода, связанного с полиморфизмом подтипов, происходит обращение к этой таблице с поиском конкретной ссылки на метод, который должен быть вызван, на основании реального типа переменной и переопределения (`override`) ею методов базового класса.

Полиморфизм подтипов реализуется через реализацию интерфейсов и через наследование, в обоих случаях класс, реализующий интерфейс может быть приведён к типу интерфейса и класс расширяющий базовый может быть приведён к базовому.

```c#
public interface ITalking
{
  public void AreUTalkingToMe() => Console.WriteLine("Yea, i'm talking to u!");
}
public class Talker() : ITalking
{
  public void AreUTalkingToMe() => Console.WriteLine("No, i'm not talking to u");
}
public class AnotherTalker() : ITalking
{
  public void AreUTalkingToMe() => Console.WriteLine("Sure, i'm talking to u!");
}

public class BasicsTalker
{
  public virtual void SayMyName() => Console.WriteLine("Whalter White");
}
public class DaughterTalker : BasicsTalker{
  public override void SayMyName() => Console.WriteLine("Haizenberg");
}

ITalking variable1 = new Talker(); // Здесь полиморфизм подтипов через реализацию интерфейса
variable1.AreUTalkingToMe(); // No, i'm not talking to u
variable1 = new AnotherTalker();
variable1.AreUTalkingToMe(); // Sure, i'm talking to u!

BasicsTalker variable2 = new BasicsTalker(); // здесь полиморфизм подтипов через наследование
variable2.SayMyName(); // Whalter White
variable2 = new DaughterTalker();
variable2.SayMyName(); // Haizenberg
```
