# Управляющие конструкции

Управляющие конструкции это специальные блоки кода, которые позволяют менять вектор вычислений программы в зависимости от условий работы кода. Условно говоря, необходимо включать полив, только при падении влажности почвы до 30%. Тогда возникает необходимость включить условие проверки и это условие и будет являться управляющей конструкцией.

- Условные конструкции
  - `if-else if-else` - условная конструкция, позволяющая менять вычисления исходя из условий

    ```csharp
    if (condition){ // Если выполняется условие
      // выполнить данный блок кода
    }else if (another_condition){ // Если 1-е условие не выполнено, но выполнено другое
      // выполнить данный блок кода
    }else { // Если ни одно из предыдущих условий не выполнено
      // выполнить данный блок кода
    }
    ```

    Минимально условная конструкция состоит из блока `if(condition)` за которым без фигурных скобок можно расположить 1 инструкцию, которая выполнится при выполнении условия, последующие инструкции выполнятся вне зависимости от выполнения условия:

    ```csharp
      int age =5;
      if (age<6)
        DoSomething(age); // Будет выполнено только если age<6
      DoAnotherSomething(age); // Будет выполнено вне зависимости от значения age
    ```

    Так же можно расположить инструкцию на той же строке, что и минимальный блок `if`: `if (age<6) DoSomething(age);`

    Блок `else if (condition)` позволяет проверить некоторое условие, если не выполнилось первое условие. Таких блоков может быть множество и каждый последующий будет проверен, если ни одно из предыдущих условий не истинно:

    ```csharp
    int age = 5;
    if (age < 1){
      DoSomething(age); // не будет выполнено, т.к. не выполнено условие
    }else if (age < 3){
      DoAnotherSomething(age); // не будет выполнено, т.к. не выполнено условие
    }else if (age < 6){
      DoAnother2Something(age); // будет выполнено, т.к. условие истинно
    }else if (age < 12) {
      DoAnother3Something(age); // Не будет выполнено, т.к. сработал предыдущий блок кода
    }
    ```

    Блок `else` позволяет задать поведение по умолчанию, если ни одно корректирующее условие не сработало:

    ```csharp
    int age = 5;
    if (age < 1){
      DoSomething(age);
    }else if (age < 5){
      DoSomething2( age );
    }else{
      NothingToDo(age); // выполнится, т.к. ни одно условие выше не выполнилось.
    }
    ```

    Условие это некоторое булево выражение, результат которого трактуется следующим образом:
    - Ложь, если результат вычислений является: `false`
    - Истина: true.

    Неявного приведения ссылочных или иных значимых типов данных к bool в C# не предусмотрено.  

    Если результат вычислений условия ложь, то последующий за if блок кода выполнен не будет и если есть другие проверки, начнут проверятся они по порядку сверху вниз, если ни одно условие не будет истинным запустится обработчик `else`, если он есть, в противном случае не будет сделано ничего.

    Помимо обычных условий есть inline условие в виде тернарного оператора `?:`:
    `переменная = (условие)? Значение_при_истине : Значение_при_ложности;`

    Применяется для читаемого заполнения некоторых переменных при простых условиях.

    Так же блок `if-else if-else` называют блоком ****ветвления****.

  - `switch-case` - так же блок ветвления, но является более оптимальным с точки зрения читабельности вариантом, если ветвлений много (более 2-3)

    ```csharp
    int age = 5;
    switch (age) {
      case 1:
        // команды
        break;
      case 2:
        // команды
        break;
      default:
        // команды
        break;
    }
    ```

    Через `switch` задаётся переменная по которой происходит сравнение, далее она сравнивается со значениями в блоке `case`, если произошло совпадение, выполняется блок кода до `break`. Ключевое слово `break` является обязательным для блока с командами, но может не указываться, если несколько `case` должны выполнить один набор команд. `default` блок выполняется после того, как ни один `case` не удовлетворил условию, т.е. ведёт себя аналогично блоку `else` из конструкции ветвления.

    С версии С# 8+ допустимы `inline switch expression` следующего вида:

    ```csharp
    string result = variable switch{
      value1 => "result1",
      value2 => "result2",
      ...
      valueN => "resultN",
      _ => "default result"
    }
    ```

    `_` - эквивалент ключевого слова `default`.

    С версии C# 9+ допустимо использовать `switch-case` с условиями вместо конкретных значений в блоках `case`:

    ```csharp
    int age = 5;
    switch (age){
      case < 4:
        // do something
        break;
      case > 4:
        // do another something
        break;
      case < 12 and > 6:
        // do another another something
        break;
      default:
        break;
    }
    ```

  - `PatternMatching` - мощный инструмент логических выражений через ключевое слово `is` позволяет манипулировать множеством вариантов проверки переменной перед блоками ветвления
    - Проверка типа

      ```csharp
        if (n is int) {
          ...
        }
      ```

    - Проверка типа с захватом значения

      ```csharp
        if (n is int p){
          // p = n
          ...
        }
      ```

    - Проверка константных значений

      ```csharp
        if (n is null){
          ...
        }
        if (n is 42){
          ...
        }
      ```

    - Всегда истинное захватывание значения (может привести к захвату null)

      ```csharp
        if (n is var p){
          // var p = n
        }
      ```

    - Проверка полей и свойств

      ```csharp
        if (obj is {Name: "John Doe", Age: >=18}){
          // Everyone John Doe elder than 18
        }
      ```

    - Проверка структуры и вложенных значений

      ```csharp
        if (obj is {Customer: {Address: {City: "Moscow"}}}){
          // Castomers from Moscow
        }
      ```

    - Деконструкция работа с кортежами и записями (Records)

      ```csharp
        if (point is (0,0)){
          Console.WriteLine("origin");
        }
        if (person is ("Jane", var age)){
          // age = person.Age and person is Jane
        }
      ```

  Условия это логические выражения булевой алгебры, расширяемые словами `and`,`not`,`or`,`xor`,`is`,`when`. Первые 4 аналогичны операторам `&&`, `!`, `||`, `^`, `is` рассмотрен выше. `xor` не может быть использован в логических паттернах, вместо него используется только побитовый `^`.

  `when` - оператор дополнительного условия.

  ```csharp
  switch (value){
    case int n when n>0: // сначала проверка на то, целый ли тип данных и захват значения, после чего проверка, больше ли нуля и только при выполнении обоих условий выполняется последующий блок кода до break.
      ...
      break;
    default:
      break;
  }
  ```

  Начиная с версии C# 11 `is` может работать со списками и массивами.

  ```csharp
  int[] arr = [1,2,3];
  if (arr is [1, _, 3]) // _ - игнорируй один элемент, остальные должны совпасть
  if (arr is [1, ..., 3]) // между 1 и 3 может быть любое число элементов, но начинаться и заканчиваться массив должен на 1 и 3 соответственно. 
  ```

  Мини-рекомендации по ветвлению:
  - `switch` для `enum` и `констант` быстрее, т.к. компилируется в таблицы-переходов.
  - `if-else` лучше использовать, когда условных конструкций мало и они короткие. Если условие большое, лучше провести декомпозицию или вынести условие в функцию. Если вариантов много, то лучше использовать `switch-case` по причине читабельности.
  - для проверок `enum` и `bool` компилятор предупредит, если в switch не все варианты рассмотрены.
  - Тернарный оператор улучшает читаемость для простых условных присвоений.

- Циклы - блоки кода, которые необходимо выполнять множество раз. Например на вход программе подаётся массив переменной длины, может прийти 5 значений, а может 100 и в таком случае используются циклы, чтобы обработка массивов данных не зависела от длины (чтобы избежать переписывания программы под длину входных данных)
  - `for` - базовый цикл со счётчиком

   ```csharp
   for (int i=0; i<10; i++){
    Console.WriteLine(i); 
   }
   // Вывод:
   // 0
   // 1
   // 2
   // ...
   // 9
   ```

   Базовый цикл со счётчиком состоит из блока подготовки к циклу, блока проверки условия, блока действий между итерациями цикла и тела цикла: `for (prepare; condition; postaction) {body}`

   В блоке `prepare`, через запятую указываются итераторные переменные и их начальные значение, но в целом, могут быть любые команды (`for (int i=0, k=0; i<N && k<2N; i++, k*=2)`). В `condition` перед каждым циклом происходит проверка условия для выполнения основного тела цикла. В `body` прописываются инструкции, которые выполняются на каждой итерации, когда условие истинно. В блоке `postaction` через запятую указываются действия, которые необходимо сделать после каждой итерации тела цикла. Обычно, там происходит изменение итераторных переменных.

   Каждый из блоков является опциональным. Т.е. код `for(;;);` вполне себе валиден и просто создаёт бесконечный цикл, который ничего не делает.

   Как и с `if` цикл `for` может быть записан без фигурных скобок, но циклично будет выполняться только `команда` на той же или следующей после `for(...)` строке. Однако, данная практика опасна, поэтому всегда надо ставить фигурные скобки. (python-клуб на этом моменте расплакался.)

  - `while` - цикл с предусловием

   ```csharp
   while(condition){
    //body
   }
   ```

   По сути аналог блока `for(;condition;)`. Важно следить за тем, чтобы `condition` не уводило `while` в бесконечный цикл. Запись `while(true);` создаёт бесконечный цикл, который ничего не делает. Запись без фигурных скобок валидна и работает так же, как `if` и `for`.

  - `do-while` - цикл с постусловием

   ```csharp
   do{
    //body;
   }while(condition);
   ```
  
   Выполняет блок кода 1 раз вне зависимости от истинности условия, после первой итерации аналогичен `while`.

Выбор между типом цикла определяется тем, что необходимо сделать. Если необходимо обойти массив - `for`, если необходимо прочитать пользовательский ввод, то `while`, если возникает задача, когда нужно что-то сделать хотя бы 1 раз, то `do-while`.

- `foreach` - цикл итерирующийся по итерируемым коллекциям

    ```csharp
    foreach(var item in collection){
     //body
    }
    ```

    `Collection` - коллекции реализующие `IEnumerable` (т.е. такую коллекцию с итерацией по ней можно реализовать самостоятельно).

    В действительности данный код разворачивается в блок следующего вида при компилции:

    ```csharp
    using (var enumerator = collection.GetEnumerator()){
     while(enumerator.MoveNext()){
       var item = enumerator.Current;
       // тело цикла
     }
    }
    ```

    Для `Span<T>` и простых массивов `foreach` генерирует оптимизированный код, который избегает перегрузок `IEnumerable`, а работает со значениями напрямую через `ref` и индексы

    Если в `foreach` попытаться изменить коллекцию, то можно получить `InvalidOperationException` в лицо.

Все циклы можно вкладывать друг в друга, если необходимо обойти, например, двумерный массив или массив массивов (массив-пила, ступенчатый массив) или есть коллекция объектов с вложенными массивами и др. ситуации. Тем не менее вложенность более 2 в одной функции считается дурным тоном с точки зрения читаемости и в идеале в каждой функции должно быть не более 1 вложенности циклов, а вложенные циклы выносить в отдельные функции.

  ```csharp
  int[,] arr = {{1,2,3},{4,5,6}};
  for(int i=0; i<2; i++){
    for (int k=0; k<3; k++){
      Console.Write(arr[i,k]);
    }
    Console.WriteLine();
  }
  ```

Так же к управляющим конструкциям относятся ключевые слова:

- `break` - перейти к следующей за блоком кода инструкции. Блоком кода считается то, что обёрнуто явно или не явно фигурными скобками. В случае циклов - выходит из них.
- `continue` - перейти на следующую итерацию цикла досрочно. Не выходит из цикла.
- `return` - завершить текущую функцию и вернуть значение. Значение может быть пустым, если функция возвращает `void`.
- `yield return` - приостановить выполнение функции и вернуть некоторое значение. Область памяти функции отправляется в кучу(`heap`) и при повторном вызове начинается с инструкции следующей за командой `yield return`

Мини рекомендации по циклам:

- `for` быстрее `foreach` из-за отсутствия вызова `MoveNext` и `Current`, тем не менее `foreach` оптимизирован для `Span<T>` и является более читаемым вариантом цикла. Если производительность не критична и не нужен точный доступ по индексам, то следует использовать `foreach`.
- `break` и `continue` в циклах можно считать антипаттерном, поскольку могут привести к спегеттификации кода, лучше декомпозировать логику. Тем не менее, ничего страшного в одном `break` или `continue` нет.
- Если в ходе цикла необходимо модифицировать коллекцию, то лучше получить список изменений (добавлений, удалений), а после применить на коллекцию через методы `RemoveAll` - для удаления или `Add` для добавления.
