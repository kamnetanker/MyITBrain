# Коллекции и структуры данных

- Массивы (`System.Array`) - непрерывный набор однотипных данных в памяти.
  
  Одномерные массивы:
  
  ```csharp
  int[] numbers = new int[5]; // Массив длиной 5 элементов, заполненный нулями ( по умолчанию через default(int) )
  int[] nums = {1, 2, 3}; // объявление массива с последующей инициализацией значениями 1, 2, 3
  int[] arr = new int[] {1,2,3}; // полная запись с указанием типа и инициализацией значениями.
  ```

  - Свойство `Length` типа `int` - возвращает размер массива (кол-во элементов в нём)
  - Для массивов, размер которых превышает `Int.MaxValue` (~2млрд) есть свойство `LongLength`
  - Индексация массивов начинается с 0, т.е. `arr[0]` - доступ к первому элементу массива.  
  - Доступ с конца массива с C# 8+ можно осуществить через `^1` - первый с конца, `^2` - второй с конца и т.д., до этой версии используется конструкция `arr.Length - 1`
  - Диапазоны (C# 8+) `int[] slice = numbers[1..3];` - вернёт элементы с индексами в интервале `[1;3)`. При выполнении операции создаёт новый массив и копирует элементы. Для `Span<T>` - представление без копирования.

  Многомерные (прямоугольные) массивы - массивы, содержащие другие массивы, но фиксированной длины. т.е. 5 массивов длины 4, например. Математическая аналогия - матрицы.

  ```csharp
  int[,] matrix = new int[3,4]; // прямоугольный массив из 3 массивов длины 4. По default типа int заполнен нулями.
  matrix[1,2] = 5; // доступ к элементу массива
  int[,] matrix_a = { {1,2}, {3,4} }; // массив из 2 массивов длиной 2
  ```

  - Все строки прямоугольных массивов одной длины
  - `GetLength(dimension)` - возврат размерности по измерению. (0 изм. - кол-во строк, 1 изм. - кол-во столбцов)
  - `foreach` - обход по порядку строк с нулевой. Т.е. сначала перебор элементов нулевой строки, потом первой и далее

  Ступенчатые/зубчатые/пилообразные массивы - массивы массивов, в которых размер массивов, входящих в исходный, может быть различным. Т.е. массив, который состоит из массивов различных длин.

  ```csharp
  int[][] jagged = new int[3][];
  jagged[0] = new int[2];
  jagged[1] = new int[4];
  jagged[2] = new int[1];
  ```

  - Каждая строка может иметь свою собственную длину, отличную от остальных.
  - Обращение к элементу `jagged[0][1]` - обращение ко второму элементу нулевой строки.
  - `foreach` не перебирает элементы вложенные в главный массив, для перебора вложенных массивов требуется вложенный цикл
  
  Статические методы массивов:

  - `Array.Sort(array)` - соритровка (требует `IComparable<T>` или передачу компаратора)
  - `Array.Reverse(array)` - разворачивание массива в обратном порядке
  - `Array.Clear(array, start, length)` - заполняет массив от `start` до `start+length-1` (т.е. диапазон `[start, start+length)`) значениями `default<T>`
  - `Array.IndexOf(array, value)` - линейный поиск

- `List<T>` - динамический массив

  ```csharp
  List<int> list = new List<int>(); // объявление  и инициализация списка. Пока он пустой, но не null.
  List<int> withCapacity = new List<int>(100); // Объявление и инициализация буфера на 100 элементов.
  List<int> fromArray = new List<int>(new int[]{1,2,3}); // Объявление и инициализация списка из массива. 
  ```

  - `Capacity` - текущий размер списка, т.е. сколько выделено памяти, может не быть равно количеству реально хранимых элементов.
  - `Count` - кол-во хранимых элементов (может не быть равно `Capacity`)
  - `Add(item)` - добавление в конец списка нового элемента (Если дошли до границы Capacity == Count, то выделяется новое место под буфер, обычно буфер удваивается)
  - `AddRange(collection)` - добавляет несколько элементов, переданных в виде коллекции, как аргумент.
  - `Insert(index, item)` - вставляет элемент на место `index` сдвигая остальные элементы в конец, занимая `O(n)` времени, где `n` - кол-во элементов массива.
  - `RemoveAt(index)` - удаляет элемент по индексу и сдвигает к началу массива оставшиеся элементы. Big-O: `O(n)`
  - `Remove(item)` - удаляет первое вхождение элемента в массив и сдвигает к началу массива оставшиеся элементы. Big-O: `O(n)`
  - `RemoveAll(predicate)` - эффективное удаление по условию. Big-O: `O(n)`
  - `IndexOf(item)` - Линейный поиск первого вхождения `item`. Big-O: `O(n)`
  - `BinarySearch(item)` - бинарный поиск первого вхождения `item`. Массив должен быть отсортирован. Big-O: `O(log n)`
  - `Sort()` - Сортировка методом быстрой сортировки. Big-O: `O(n * log n)`

  Производительность операций:
  
  - Доступ по индексу: `O(1)`
  - Добавление в конец: амортизировано `O(1)`. Худший случай, если будет аллокация - `O(n)`
  - Вставка/удаление в середине: `O(n)`

  Рекомендация: если известен порядок элементов с которым предполагается работа, можно выделить место сразу, чтобы избежать лишних копирований элементов при расширении массива.

- `Dictionary<TKey, TValue>` - Хэш-таблица. Набор пар: уникальный ключ : значение
  
    ```csharp
    Dictionary<string, int> dict = new Dictionary<string, int>(); // Объявление и инициализация хэш-таблицы
    dict["one"]=1; // через индексатор можно добавить или перезаписать значение
    dict.Add("two", 2); // Через Add(key, value) если индекс уже есть в хэш-таблице, то выкинется исключение
    ```

    Методы:

  - `TryGetValue(key, out var value)` - поиск значения по ключу, не требует проверки наличия ключа в хэш-таблице.
  - `ContainsKey(key)` - проверка наличия ключа в таблице. Big-O: averege `O(1)`
  - `ContainsValue(value)` - проверка наличия значения в таблице. полный перебор. Big-O: `O(n)`
  - `Remove(key)` - Удаление пары ключ:значение. Big-O: `O(1)`
  - `Keys` и `Values` - коллекции без гарантии порядка.

    Принцип работы:

    1. Вычисляется хэш-код ключа
    2. Берётся остаток от деления по модулю длины `buckets`
    3. В корзине `buckets` индекс которой был получен проверяется совпадение хэш-кодов, после чего ключи сравниваются через `Equals()`, если совпали хеш-коды;
    4. При добавлении, если корзина переполнена, происходит увеличение таблицы и перехэширование содержимого.

    Требования к ключам:  

    1. Объекты ключи - иммутабельны. Необходимо для того, чтобы ключи не меняли свои хэш коды, это важно.
    2. GetHashCode() и Equals() должны быть консистентны
    3. Хэш после помещения в хранилище не должен меняться.

    Добавление/поиск/удаление Big-O: avg `O(1)`, worst `O(n)` (коллизии, перехэширование)

- `HashSet<T>` - множество уникальных элементов.
  
    ```csharp
    HashSet<int> set = new HashSet<int>(); // Объявление и инициализация хэш-сета

    bool added = set.Add(1); // true
    added = set.Add(1); // false
    ```

  Операции:

  - `UnionWith(other)` - Объединение множеств
  - `IntersectWith(other)` - Пересечение множеств
  - `ExceptWith(other)` - Удалить элементы, входящие в other из текущего множества
  - `SymmetricExceptWith(other)` - В множестве останутся только элементы присутствующие в одном из множеств, а те, что есть в обоих удалятся.
  - `Contains(item)` - проверить наличие элемента в множестве. Big-O: `O(1)`
  - `Remove(item)` - Удалить элемент из множества. Big-O: `O(1)`
  
  Используется для удаления повторяющихся элементов из коллекций и операций над множествами.

- Очередь и Стек (`Queue<T>`,`Stack<T>`)
  
  `Queue<T>` - FIFO очередь.
  
    ```csharp
    Queue<int> queue = new Queue<int>(); // Объявлени и инициализация очереди

    queue.Enqueue(1); // Помещение элемента в очередь

    int peek = queue.Peek(); // Получить элемент из очереди, но не удалять его в очереди
    int dequeue = queue.Dequeue(); // Получить элемент из очереди и удалить его в очереди
    ```

  `Stack<T>` - LIFO очередь.

    ```csharp
    Stack<int> stack = new Stack<int>(); // Объявлени и инициализация стека
    stack.Push(1); // Помещение элемента в стек
    int peek = stack.Peek(); // Аналогично Queue<T>
    int top = stack.Pop(); // Берёт верхний элемент стека, возвращает его и удаляет из стека
    ```

  Обе очереди реализованы на основе массива с динамическим расширением.
  
  - `Count` - количество элементов
  - `Clear()` - очистить очередь

- `IEnumerable<T>` и `IEnumerator<T>` - паттерн итератора.

    `IEnumerable<T>` - интерфейс. Классы, реализовавшие этот интерфейс предоставляют итераторы (`IEnumerable<T>`) для `foreach` и не только:

    ```csharp
    IEnumerator<T> GetEnumerator();
    ```

    `IEnumerator<T>` - интерфейс. Класс, реализующий данный интерфейс предоставляет методы для обхода некоторой коллекции, будь то коллекция из стандартных библиотек, или коллекция, которую программист реализовал самостоятельно. Так же порядок обхода можно переопределить и для ста

  - `T Current` - текущий элемент, на который указывает итератор.
  - `bool MoveNext()` - переход к следующему элементу коллекции
  - `void Reset()` - Устаревший метод, вызывает исключение `NotSupportedException`, поскольку считается `Deprecated`.  
  - `void Dispose()` - освобождение ресурсов.

    `foreach` работает с любой коллекцией, для которой есть реализованный `IEnumerable<T>`.

    Для `Span<T>` foreach генерирует оптимизированный код, не используя `IEnumerator<T>`

    `yield return` фактически реализуется через итератор, но без промежуточной коллекции. Его нельзя использовать внутри `catch` и `finally`, только в `try`.

- `Immutable-коллекции` - коллекции, для который при изменении содержимого создаётся полная копия коллекции с внесёнными изменениями. Потокобезопасны без блокировок, поскольку область памяти коллекции не меняется при внесении изменений, а выделяется новая область, где копия коллекции с внесёнными изменениями.

    Доступны через `System.Collections.Immutable`, который может потребовать установки дополнительного `NuGet-пакета`

    Типы данных:

  - `ImmutableArray<T>` - В целом, просто массив, просто при изменении копируется полностью.
  - `ImmutableList<T>` - реализован через двоичное дерево и потребляет больше памяти, но скорость изменения `O(log n)`
  - `ImmutableDictionary<TKey, TValue>`, `ImmutableHashSet<T>`

    В целом, аналогичны своим мутабельным версиям по использованию. Но для эффективной работы рекомендуется создавать билдеры и из билдеров набирать набор мутаций, которые после применять на коллекцию, что существенно повышает итоговую скорость работы с иммутабельными коллекциями.

    ```csharp
    var builder = ImmutableArray.CreateBuilder<int>();
    builder.Add(1);
    builder.Add(2);
    ImmutableArray<int> array = builder.ToImmutable();
    ```
