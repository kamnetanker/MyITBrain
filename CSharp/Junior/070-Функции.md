# Функции

- Основы

    Написание всего кода внутри функции `Main(string[] argv)` привела бы к тому, что всё программное обеспечение со временем превращалось бы в неподдерживаемое, спагеттифицированное, постоянно дублирующееся полотно нечитаемого текста. Чтобы избежать этого существуют функции - именованный набор команд, обрабатывающих входные данные и возвращающие некоторый результат вычислений (а могут и не возвращать - `void`). Функции так же именуют словом "метод", если они являются частью класса.  

    Общий вид метода выглядит следующим образом:

    ```csharp
      [access_modifier] [static] ReturnType<T> MethodName(ParamType param){

        return ReturnType<T>;
      }
    ```

    где:

  - `access_modifier` - необязательный модификатор доступности: `public`, `private`, `protected`. По умолчанию метод является `private` и доступен только для экземпляров класса, а `public` доступен в любом месте кода. О модификаторах доступа подробнее в блоке о классах
  - `static` - модификатор, определяющий то, работает ли данный метод с полями и значениями экземпляров класса или нет. Статические методы не работают с полями и методами экземпляров и вызываются `ClassName.StaticMethod`, как пример `Math.Pow(...)`. `Pow(...)` - статическая функция класса `Math`. Более подробно про статически классы и их статические методы будет в блоке про классы. Так же статическим методом является метод `Main()`.
  - `ReturnType<T>` - некоторый возвращаемый тип данных методом. Может быть void, если метод ничего не возвращет. Если метод что-то возвращает, то в теле метода обязательно необходимо указать конструкцию `return someValue;`, где `someValue` будет типом, неявно приводимым к возвращаемому. Если метод ничего не возвращает, то `return` указывать не обязательно, но если необходимо, то никаких значений он не должен возвращать: `return;`
  - `MethodName` - имя метода. По соглашению о именовании, методы именуются в нотации `CamelCase`. Подробнее о соглашениях по именованию разных вещей в C# будет в блоке про классы.
  - `ParamType param` - некоторый набор параметров, передаваемых функции и с которыми она должна выполнить некоторые вычисления. Таких пармаетров через запятую может быть указано сколько угодно, а так же можно использовать ключевое слово `params` перед типом данных передаваемого параметра, если есть необходимость в том, чтобы метод принимал переменное количество параметров этого типа данных. `params int[] numbers` - примет сколько угодно чисел или массив чисел. `params` указывается только один раз и после всех остальных параметров, переменное количество параметров может быть только у одного параметра.
  - Далее идёт тело метода и `return` возвращающий результат того типа данных, который был объявлен перед именем метода. В теле метода могут вызываться другие методы и он сам(рекурсия), производится любые вычисления, создаваться переменные, экземпляры классов, лямбда-выражения и анонимные структуры данных.

- Рекурсия

  Ситуация в которой метод вызывает сам себя или группа методов циклично вызывают друг друга, увеличивая глубину вложенности вызовов - называется рекурсия.

  Классический пример рекурсионного метода - вычисление факториала, чисел Фибоначчи и других числовых последовательностей с различными законами, где следующее вычисляемое значение зависит от ближайших предыдущих.

  Главный минус рекурсивных алгоритмов, реализованных таким образом, что методы вызывают сами себя - высокий расход памяти в стеке вызовов. Каждый экземпляр исполняемого методы имеет свою "страницу" памяти, которая помещается в стек вызовов каждый раз, когда эта метод вызывает другой метод или самого себя. Этот стек ограничен, обычно 1-4МБ данных, что ориентировочно равно 1000-2000 вызовов, в зависимости от размера страницы данных самого метода, который определяется созданными переменными и количеством инструкций в методе.

  Так же C# не гарантирует потимизацию "хвостовой рекурсии" - особого частного случая рекурсии, при котором рекурсивный вызов является последней операцией в теле метода и гарантированно может быть преобразован в итеративный алгоритм, но C# такого не гарантирует, что может привести к неожиданным ошибкам или перерасходу ресурсов.

  Рекурсивных алгоритмов, реализованных через вызов методом самого себя или замкнутого цикла вызовов из нескольких методов следует избегать. Вместо этого следует реализовывать рекурсивные алгоритмы, через циклы, обрабатывающие кадры контекста (некоторые структуры данных, содержащие информацию для работы алгоритма), которые на каждой итерации помещаются в стек, хранимый в куче, а не в стеке вызовов.

  Пример алгоритма, реализованного через рекурсивный вызов:

  ```csharp
  int Factorial(int N){
    return N*Factorial(N-1);
  }
  ```

  А здесь пример, реализованный через стек:

  ```csharp
  int StackFactorial(int N){
    Stack<int> stack = new Stack<N>();
    for(int i=N; i>1; i--){
      stack.Push(i);
    }
    int result=1;
    while(stack.Count>0){
      result *= stack.Pop();
    }
    return result;
  }
  ```

  Пример не самый оптимальный, но как демонстрация - пойдёт. В действительности, алгоритм вычисления факториала реализуется через цикл:

  ```csharp
    int CycleFactorial(int N){
      int result = 1;
      for(int i=N; i>1; i--) {
        result*=i;
      }
      return result;
    }
  ```

  Но с задачей демонстрации того, что рекурсивный алгоритм можно реализовать через контекстный стек - пример справляется.

  Алгоритм, позволяющий развернуть рекурсивный алгоритм в итерацию по стеку:
  
    1. Определить данные, помещаемые в стек вызовов при каждом рекурсивном вызове
    2. Поместить данные в стек
    3. Определить операции, выполняющиеся после возврата из функции
    4. Выполнить операции с содержимым стека.
    5. Profit.

  графомания получилась лютая.

- Передача по ссылке и значению

  Параметры в метод могут передаваться по ссылке на объект и по значению. По значению передаются значимые типы данных: `byte`, `short`, `char`, `int`, `long`, `float`, `double` и их модификации.

  По умолчанию в C# все аргументы передаются по значению, т.е. для значимых типов происходит копирование их значений, а для ссылочных типов данных (все экземпляры классов - объекты) передаются копии ссылок, но сам объект не копируется.

  Для управления передачей по ссылке есть модификаторы: `ref`, `out`, `in`.

  - `ref` - аргумент, передаваемый с этим модификатором должен быть инициализирован до вызова метода. Внутри метода можно читать и изменять переданный аргумент и изменения затронут исходный экземпляр, ссылка на который передаётся.
  - `out` - передача значения наружу. Переменная не обязана быть инициализирована до вызова метода, но в методе до возврата должна быть присвоена.
  - `in` - передача по ссылке с запретом на запись. Аналогично `ref readonly`.

  Рекомендации:

  - Модификаторы лучше использовать, чем полагаться на по умолчанию. Поскольку код с явно указанными модификаторами лучше читается
  - `out` - лучший вариант для возврата множества значений. Основной возврат - `bool` флаг успешности операции и возврат значения через `out`.
  - `ref` - лучше использовать, когда требуется взаимодействие с большим по объёму количеством данных, которые нецелесообразно копировать для изменения. Но большое количество `ref` лучше заменить.
  - для `struct` более 16 байт целесообразно для экономии памяти использовать `in`

- Необязательные параметры и именованные аргументы

  Необязательные параметры - параметры, для который существуют значения по умолчанию. Такие параметры необходимо указывать ПОСЛЕ обязательных. Так же любой параметр можно передать именованным образом, не в том порядке, в котором оно указано:

  ```csharp
  void Log(string message, string level = "INFO"){ ... }
  Log("Started"); // Параметр level имеет значение по умолчанию, потому не обязательно его указывать, он примет то значение, которое указано в объявлении функции
  Log("Wet", "Warning"); // параметр level заменяет своё значение на Warning
  Log(level: "Error", message: "No data recieved"); // здесь аргументы передаются не позиционно-зависимым методом, а поимённо
  ```

- Локальные функции

    Методы, объявленные внутри других методов являются локальными и недоступны извне этого метода. Могут использовать локальные переменные и параметры метода в котором объявлены. Следует использовать локальные функции вместо лямбд тогда, когда можно обойтись без лямбда-функций(лямбда-выражений). Т.е. тогда, когда не требуется использовать делегаты, например. Исключения, возникшие внутри локальных методов выбрасываются сразу, а не отложенно, как в лямбдах или итераторах.

    ```csharp
    void ProcessData(int[] data)
    {
        if (data == null) throw new ArgumentNullException(nameof(data));

        double result = CalculateAverage();    // вызов локальной функции
        Console.WriteLine(result);

        double CalculateAverage()
        {
            if (data.Length == 0) return 0;
            return data.Average();
        }
    }
    ```

- Возврат значений. Кортежи и деконструкция.

  Возврат из метода осуществляется ключевым словом `return`, после чего по стеку вызовов исполнение возвращается на следующую за вызовом команду. Если вызов был справа от присвоения или выражения с математическими операторами или другими командами, то выполнение продолжится со следующей по очерёдности операции.

  Методы с возвращаемым типом `void` не возвращают никакие значения. Остальные методы возвращают то значение, которое стояло справа от ключевого слова `return` и до символа конца инструкции. Если там было выражение, то вернётся результат этого выражения. пример: `return 25*3-9;` - вернёт тип `int`, т.е. метод должен быть объявлен с возвращаемым типом в который неявно можно преобразовать `int` либо непосредственно `int` и вернётся значение 66.

  С C# 7.0 в качестве возвращаемого значения может указываться кортеж - легковесная структура данных с несколькими значениями. Пример использования показан ниже:

  ```csharp
    (int Sum, int Count) Calculate (int[] numbers){
      return (numbers.Sum(), numbers.Length);
    }
    var result = Calculate(new int[]{1,2,3});
    // result.Sum = 6, result.Count = 3;
    var (sum, count) = Calculate(new int[] {1,2,3}); // Деконструкция - элементы кортежа "распаковываются" по переменным sum и count. Названия не важны, важен порядок указания деконструкции.
    // (var sum, var count) - тоже вариант деконструкции 
  ```

  Если требуется вернуть пару-тройку значений, то использовать кортеж - ок. Для большого количества возвращаемых значений, тем более разных типов, лучше использовать `record`, `class`, `struct`.

- Статические методы

    Статические методы принадлежат типу данных, а не экземпляру класса в которых объявлены. Как пример: `Math.Sqrt(25);`

    Лучшие практики:

    1. Если для функционирования метода ему достаточно передаваемых аргументов и он не взаимодействует с полями класса, то он должен быть статическим
    2. Статические методы и классы усложняют тестирование и отладку программ.

- Модификаторы доступа - контроль того, откуда может быть вызван метод или откуда может происходить обращение к полям класса.

  - `public` - доступ откуда угодно, кем угодно
  - `private` - только изнутри того же класса/структуры
  - `protected` - изнутри класса и наследников
  - `internal` - внутри текущего экземпляра
  - `protected internal` - внутри текущего экземпляра или из наследников в другом экземпляре
  - `private protected` - изнутри экземпляра и только наследников

- Перегрузки

    Перегрузка метода - несколько методов, имеющих одинаковое имя, но различное количество и тип данных аргументов, возвращаемый тип и могут иметь различную логику работы.

    ```csharp
    int Add(int a, int b) => a+b;
    double Add(double a, double b) => a+b; // перегрузка, где используется иной тип данных
    ```
