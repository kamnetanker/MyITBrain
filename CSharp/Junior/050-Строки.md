# Строки и форматирование

- Основы строк.

    Строка - иммутабельный ссылочный тип данных, содержащий массив символов UTF-16. Следовательно, любая операция, изменяющая содержимое строки приводит к созданию нового объекта с копированием содержимого старого в новый. Это очень важно для понимания производительности операций со строками.

- Форматирование строк

  - `String.Format(string[, params...])` - предоставляет возможность подставлять собственные значения в строки, на основе специального формата: `{index[,aligment][:formatString]}`
    - `index` - номер переменной слева направо, указанной в параметрах при вызове метода `Format(...)` пример: `String.Format("{1} {0}", value_0, value_1);` - value_1 будет подставлено перед value_0, поскольку первым в строке указан индекс 1.
    - `aligment` - выравнивание подставляемого значения, читай "ширина столбца". Т.е. если мы выводим что-то длиной 3 символа с выравниванием 10, то перед значением подставится 7 пробелов, если с выравниванием -10, то после подстановки вставится 7 пробелов.
    - `formatString` - формат подставляемой строки, например, чтобы вывести дробное число. можно использовать формат "F2", что при выводе обрежет дробную часть 2 символами. Пример: `String.Format("{0:F2}",3.14159); // Вывод: 3.14`  
      Так же можно форматировать дату, указывая что нужно вывести только часы, или только минуты: `:hh`,`:mm` и другие вариации `{0:yyyy-MM-dd hh:mm:ss}` - `2026-06-10 09:28:13` как пример.

- Интерполяция строк - Синтаксический сахар для `String.Format(...)`, при написании разворачивается компилятором в `String.Format(...);`

  ```csharp
  Console.WriteLine($"{DateTime.Now,25:yyyy-MM-dd hh:mm:ss}");
  // Вывод: "      2026-06-10 09:32:37" пробелы спереди добавлены из-за выравнивания
  ```

  Указывать выравнивание и формат не обязательно, можно просто выводить переменные или литералы.

  Можно использовать `$$""` и более знаков `$`, чтобы указать, сколько фигурных скобок используется для обозначения подставляемого выражения (C# 11+):

  ```csharp
  Console.WriteLine($$"{Datetime}:{{DateTime.Now,25:yyyy-MM-dd hh:mm:ss}}");
  // Вывод: "{Datetime}:      2026-06-10 09:32:37"
  ```

  Так же с C# 11 версии можно использовать многострочную интерполяцию.

  С версии C# 10 и .Net 6+ Microsoft ввели механизм InterpolatedStringHandler, который позволяет перехватывать обработку интерполяции и преобразовывать её в StringBuilder повышая производительность множественных интерполяций.

- `StringBuilder` - рекомендуемый способ сборки строк в циклах.

    ```csharp
    using System.Text;
    //....
    StringBuilder sb = new StringBuilder();
    for(int i=0; i<10; i++){
      sb.Append(i.ToString());
    }
    string resultStr = sb.ToString(); 
    Console.WriteLine(resultStr);
    // Вывод: 0123456789
    ```

    Для разовых конкатенаций разумно использовать обычные операции `+` или интерполяцию, если же конкатенаций много, лучше переработать алгоритм так, чтобы собирать строки через `StringBuilder`. Так же для StringBuilder можно задать исходный размер буфера, чтобы избежать излишних переаллокаций, если заранее известен количественный порядок строк, которые требуется собрать: `new StringBuilder(256);`

- `string.Join(...)` - позволяет объединить эффективно массив строк, список строк или любую другую итерируемую коллекцию строк.
  
    ```csharp
    string[] words = {"Hello", "world", "from", "C#"};
    string resultStr = string.Join(" ", words); // Первым аргументом указывается разделитель итоговых строк.
    Console.WriteLine(resultStr);
    // Вывод: Hello world from C#
    // Если используется одиночный символ-разделитель, то перегрузка с char работает быстрее.
    ```

- `string.Split` - разбиение строк по символу-разделителю

    ```csharp
    string fullString = "Hello world from C#";
    string[] splittedArr = fullString.Split(' '); // Через запятую можно указывать более 1 разделителя. А так же можно указать массив сепараторов типа char в качестве первого аргумента
    // splittedAdd = {"Hello", "world", "from", "C#"}; 
    ```

    С помощью `StringSplitOptions` можно задать поведение: `Split(Char[], StringSplitOptions)`
      *Удалить пустые вхождения: `StringSplitOptions.RemoveEmptyEntries`
      * Обрезать пустые символы: `StringSplitOptions.TrimEntries`
      * Использовать оба, просто укажите через вертикальную черту: `StringSplitOptions.RemoveEmptyEntries | StringSplitOptions.TrimEntries`

- Сравнение строк.

    Операторы `==` и `!=` используют побайтовое сравнение, игнорируя возможное логическое равенство строк. Например для стандарта URL допустимо, что ссылка может быть написана как в верхнем, так и в нижнем регистре, а побайтовое сравнение может показать, что строки не равны, даже если регистр отличается у 1 символа.

    Для решения этой проблемы существует `using System.Globalisation;` - Библиотека для работы с разными культурами и перечисление StringComparison - указывающее на то, как методы, осуществляющие сравнение должны работать с регистром и некоторыми культурными особенностями текстов.

  - `Ordinal` - побайтовое сравнение строк с учётом регистра, эквивалентно работе операторов сравнения `==` и `!=`
  - `OrdinalIgnoreCase` - побайтово без учёта регистра
  - `CurrentCulture` - сравнение на основе текущей системной культуры и её особенностей
  - `CurrentCultureIgnoreCase` - то же, но без учёта регистра.
  - `InvariantCulture` - сравнение на основе искусственной культуры на основе английского языка и не зависит от системных настроек
  - `InvariantCultureIgnoreCase` - то же, но без учёта регистра

    Вышеуказанные настройки могут использоваться в методах, рекомендуемых для работы со строками:

  - `stringValue.Equals(string? value, StringComparison comparisonType)` и статический вариант `String.Equals(string? value1, string? value2, StringComparison comparisonType)` - сравнивают равенство строк, возврат true - строки равны
  - `String.Compare (string? value1, string? value2, StringComparison comparisonType)` и `stringValue.CompareTo(string value2)` - используется для упорядочивания строк. возврат <0 указывает на то, что первый операнд должен следовать прежде, чем второй операнд при сортировке по возрастанию `Console.WriteLine(String.Compare("апельсин", "яблоко", StringComparison.Ordinal)); //-31`
  - `stringValue.Contains(string value, StringComparison comparisonType)` - проверка, входит ли строка value в строку stringValue. `True` - если входит.
  - `stringValue.IndexOf(string value, StringComparison comparisonType)` - индекс первого вхождения value в stringValue. Аналогично `stringValue.LastIndexOf(...)` - находет индекс последнего вхождения. Возврат -1 значит, что ничего не найдено.
  - `stringValue.Replace(string oldvalue, string? newvalue, StringComparison comparisonType)` - замена старых значений новыми. comparisonType позволяет, например, заменить игнорируя регистр.
  - `stringValue.StartsWith(string? value, StringComparison comparisonType)` и то же для `EndsWith` - позволяет проверить начинается ли строка с указанного значения или нет.

- `String.Create()` и `Span<T>`
  
    `String.Create()` - метод создания строк, который сразу выделяет память, необходимую для финальной строки, но посчитать эту память - задача программиста.

    Сигнатура выглядит следующим образом:

    ```csharp
      public static string Create<TState> (int length, TState state, SpanAction<char, TState> action);
    ```

    `length` - длина итоговой строки  
    `tstate` - любые данные, которыми заполяется строка. Можно передавать кортежем.  
    `action` - делегат, задача которого - заполнить созданную строку данными из `state`

    ```csharp
    string user = "John";
    string domain = "example.com";

    string email = string.Create(user.Length + domain.Length + 1, (user, domain), static (span, state) =>
    {
        // Копируем имя пользователя в начало span'а
        state.user.AsSpan().CopyTo(span);
        // Устанавливаем символ '@' на следующую позицию
        span[state.user.Length] = '@';
        // Копируем домен, начиная с позиции после '@'
        state.domain.AsSpan().CopyTo(span.Slice(state.user.Length + 1));
    });
    Console.WriteLine(email); // Вывод: John@example.com
    ```

    `Span<T>` - структура на стеке, которая не создаёт данные и не копирует их, а указывает на исходные области в куче. Из-за того, что эта структура не существует в куче - нельзя создать её как поле класса, нельзя использовать в асинхронных методах и нельзя использовать в итераторах `yield return`

    `Span<T>` - высокопроизводительная структура, но она требует, чтобы память была заранее размечена.

    Есть `ReadOnlySpan<T>` - неизменяемая версия.

    К `Span<T>` можно привести массивы и списки.

    Операции со `Span<T>`:

    1. `span.Slice(index, length)` - возвращает новый span, который смотрит в указанное окно индекс + длина. Данные не копируются и остаются исходными.
    2. `span.CopyTo(anotherSpan)` - копирование данных в область, на которую смотрит спан
    3. `span.Fill(T variable)` - заполнение области, в которую смотрит спан значением переменной
    4. Работа с индексами `span[index]` - работа непосредственно по месту.
