# Базовые математические операции

- Арифметика  
  Базовые арифметические операции с ***ЦЕЛЫМИ ЧИСЛАМИ***
  - `+`, `-`, `*` - выполняются как обычные математические операции.  

    ```csharp
    Console.WriteLine(7+6-3);
    // Вывод: 10
    Console.WriteLine(7*8);
    // Вывод: 56
    ```

  - `/` - операция деления выполняется до целого, т.е. при выполнении операции a/b вернёт целое количество вхождений числа b в число a. Например 12/4 выведет 3, поскольку число 4 можно 3 раза уместить в 12. Для 13 и 14 то же самое, число 4 только 3 целых раза можно уложить в 13 и 14.

    ```csharp
    Console.WriteLine(12/4);
    // Вывод: 3
    Console.WriteLine(14/4);
    // Вывод: 3 (Полное деление будет 3.5, но дробная часть отбрасывается и результат - только целая часть)
    ```

  - `%` - остаток от деления (деление по модулю (напр. 12%5 есть остаток от деления 12 по модулю 5))

    ```csharp
    Console.WriteLine(12%5);
    // Вывод: 2
    ```

  - `++`, `--` - операторы инкремента и декремента соответственно. В зависимости от расположения перед или после аргумента меняют свой порядок выполнения.

    ```csharp
    int a=5;
    Console.WriteLine(a++);
    // Вывод: 5 Значение a после Console.WriteLine будет 6. 
    Console.WriteLine(a);
    // Вывод: 6, Значение a не изменилось
    Console.WriteLine(++a);
    // Вывод: 7, Значение a стало 7 до того, как a передан в функцию WriteLine
    ```

  Переменные имеют ограниченный размер с максимальными значениями, которые можно получить введя `<type>.MaxValue` (напр. `int.MaxValue`), при попытке добавить ещё что-то к этому числу, произойдёт переполнение, что не вызовет ошибки в программе, но значение станет минимальным для данного типа данных.  
  Пример для `byte` и `sbyte`:  

  ```csharp
  byte a = 255; // В бинарном виде 8 единиц: 0b11111111
  a=a+1; // Происходит переполнение и в памяти образуется число: 0b100000000(единица и 8 нулей), после чего происходит запись в 8-ми битовое (т.е. 1-но байтовое) пространство переменной а в результате чего остаются только 8 нулей с конца результата и значение a становится 0.
  Console.WriteLine(a);
  // Вывод: 0
  sbyte b = sbyte.MaxValue; // 0b01111111 = 127
  b=b+1; // 0b10000000 = -128
  Console.WriteLine(b);
  // Вывод: -128
  ```  

  Для контроля за переполнением используется конструкции `checked` и `unchecked`, которые позволяют включать и отключать проверку переполнения для переменных. Для не константных значений проверка отключена по умолчанию, для константных включена и для `decimal`

  ```csharp
  int a = int.MaxValue;
  try{ // Обёртка для участка кода, который может вызвать исключение
    checked{ // Создаём блок кода, в котором необходимо осуществлять проверку переполнения переменных
        a+=1; // Переполняем переменную, вызывая исключение
    }
  }
  catch(Exception e){ // Получаем исключение
    Console.WriteLine(e.Message); // Выводим сообщение об исключении.
  }
  // Вывод: Arithmetic operation resulted in an overflow.
  ```

    Базовые арифметические операции с ***ДРОБНЫМИ ЧИСЛАМИ***  
    В базовом приближении операции `+`, `-`, `*`, `/` работают так же, как в обычной математике, т.е. учитывая дробные остатки. (6/4 вернёт 1.5, 4*1.5 вернёт 6), однако ввиду бинарной природы компьютерных мозгов всегда есть математическая ошибка вычислений. Данную ошибку называют `epsilon` и связана она с тем, что в двоичном виде десятичные дроби не выражаются точно. Например число 0.3 в двоичном виде записывается с ошибкой и имеет в конце, после множества нулей 4-ку. т.е. `0.1+0.2` будет `0.30000000000000004`. Данный факт следует учитывать, поскольку при последовательном выполнении множества операций с плавающей точкой ошибка может накапливаться и приводить к ошибкам в вычислениях. Для минимизации таких ошибок следует пользоваться типами данных, которые "под капотом" считают в десятичной системе счисления, например `Decimal`. Т.е. финансы лучше считать через `Decimal`, хоть это и сильно медленнее.  
    Для сравнения дробных чисел следует использовать некоторое малое число `epsilon` для типа `double` его можно получить через `double.Epsilon`, которое вводится как переменная.  

    ```csharp
    double epsilon = double.Epsilon * 1e2; //условно минимальное значимое число для double и поднять его порядок на 2
    double a=0.1d;
    double b=0.2d;
    if(Math.Abs(a+b-0.3d)<epsilon) {
        Console.WriteLine("Truly cinema");
    }else{
        Console.WriteLine("Not Cinema");
    }
    // Вывод: Truly cinema
    ```  

    Так же при работе с дробными числами следует помнить следующее:
  - Попытка деления на 0 не вызовет ошибку, в число будет записано `NaN` т.е. *Not a Number*, за исключением целочисленных типов, там будет ошибка `DivideByZeroException`
  - Деление на +0 или -0 ошибку не вызовет, но в числе будет записано `PositiveInfinity` или `NegativeInfinity` соответственно.
  - Для `decimal` деление на ноль вызывает ошибку `DivideByZeroException`.
  - `Math.Sqrt(-1.0)` так же вернёт `NaN`
  - При выполнении математических операций между различными типами данных происходит неявное преобразование типов, таблица которого приведена ниже. Так же есть запрещённые неявные преобразования (например `ulong` и `long` не могут быть неявно приведены друг к другу в рамках математической операции)
  - При одновременном выполнении математической операции между знаковым и беззнаковым числом результат будет знаковым и большей размерности. Например:

      ```csharp
        byte a= 200;
        sbyte b= -50;
        Console.WriteLine((a+b) is int);
        // Вывод: true
      ```

- Преобразование типов данных

    Таблица неявных числовых преобразований показывает какие типы данных без явного указания типа могут быть приведены к другим типам данных.

    ```markdown
    | От (from) | К (to)                                                      |
    | _________ | ___________________________________________________________ |
    | sbyte     | short, int, long, float, double, decimal, nint              |
    | byte      | short, ushort, int, uint, long, ulong, float, double, decimal, nint, nuint |
    | short     | int, long, float, double, decimal, nint                     |
    | ushort    | int, uint, long, ulong, float, double, decimal, nint, nuint |
    | int       | long, float, double, decimal, nint                          |
    | uint      | long, ulong, float, double, decimal, nuint                  |
    | long      | float, double, decimal                                      |
    | ulong     | float, double, decimal                                      |
    | float     | double                                                      |
    | nint      | long, float, double, decimal                                |
    | nuint     | ulong, float, double, decimal                               |
    ```

    Важно:

  - Расширение типа не всегда безопасно. `int` -> `long` (безопасно) `int` -> `float` (не безопасно), а `long` -> `float` уже потеря точности.
  - Дерева преобразования нет, есть конкретные правила преобразования.

    Процесс приведения типов при выполнении математических бинарных операций:

    1. `ulong` с операндами `sbyte`, `short`, `int`, `long` - ошибка компиляции
    2. `ulong` с другими операндами приводит к результату `ulong`, второй операнд приводится к типу `ulong`
    3. `long` с другими операндами приводит к результату `long`, второй операнд приводится к типу `long`
    4. `uint` с операндами типа `sbyte`, `short`, `int` приводит к приведению обоих до типа `long` и результат `long`
    5. `uint` с другими операндами приводит к приведению этих операндов к `uint` и результат `uint`
    6. Если условия выше не выполнены, то оба операнда к `int`.

- Булева арифметика  
    При работе с базовыми типами данных(и не только) можно использовать побитовые операции, меняя содержимое с контролем управления вплоть до бита данных(для этого и необходимо знать размерность чисел, например), а так же с помощью булевых операций можно производить логические вычисления (для условных конструкций, например)  
    `Побитовые операции` - выполняются с каждым битом предоставленного операнда или операндов попарно, т.е. 1-й бит операнда слева с 1-м битом операнда справа, потом со 2-м битом каждого операнда, потом с третьим и т.д.:
  - `&` побитовое AND - если операнд слева и справа 1, то результат 1. 0b1011 & 0b1010 = 0b1010 (операция выполняется для каждого бита, два конечных не выполнили условие для получения 1 и результат последнего бита стал 0)
  - `|` побитовое OR - если операнд слева или справа 1, то результат 1. 0b1011 | 0b1010 = 0b1011 (последний бит у левого операнда был 1, поэтому в результате 1)
  - `^` побитовое исключающее или. Если операнд слева отличается от операнда справа, то единица 0b1011 ^ 0b1010 = 0b0001 (отличался только последний бит)
  - `~` побитовая инверсия. Каждый бит операнда справа инвертируется (унарная операция)  ~0b1011 = 0b0100
  - `<<` и `>>` - операции сдвига. Сохраняют знак, остальное заполняют нулями. 0b101 << 2 = 0b10100, 0b101 >> 2 = 0b001
  - `>>>`  С# 11 - сдвиг с сохранением знака, кроме знака заполняет остальное нулями.
    `Логичесике операции` - выполняются с парами операндов (или одним, если унарные), истиной является `true`, а `false` или пустота как ложь, 1 и 0 соответственно. Результат вычилений либо истина либо ложь.
  - `&&` - логическое И - истинно, когда операнд слева и справа истинны.
  - `||` - логическое ИЛИ - истинно, когда хотя бы один операнд слева или справа истинный.
  - `!` - логическая инверсия. Если операнд справа истина, то делает ложью и наоборот
  - `^` - логический оператор исключающего ИЛИ. Истина, если операнды слева и справа отличаются.
  - `<`,`>`,`<=`, `>=` - логические операторы сравнения, работают аналогично классической математике.
- `Decimal` - численный тип данных для финансовых вычислений. В целом, работа с ним мало отличается от `double`, тем не менее несколько нюансов надо знать.  
  - При делении сохраняется масштаб и числа приводятся к `Decimal` 28/5m -> 5.6
  - Остаток от деления аналогичен double, но точность выше.
  - Округление требуется задавать явно.  `Math.Round(value, 2, MidpointRounding.AwayFromZero);`
  - При делении на 0 будет ошибка, как и при работе с целыми числами
  - `NaN` не применим, будет ошибка
  - Переполнение ВСЕГДА вызовет ошибку, даже в блоке `unchecked`
- Строки и символы.
  - Символы `char` неявно преобразуются в `ushort`, поэтому вся арифметика `ushort` работает и здесь. При вывода будет отображен символ `UTF-16` соответствующий числу по таблице символов указанной кодировки.
  - Строковые операции `string`. Строки иммутабельны, любые модификации приводят к созданию НОВЫХ строк, это обязательно надо помнить, поскольку это максимально влияет на производительность, например для конкатенации строки длиной N и строки длиной N1 будет израсходовано 2(N+N1) памяти и N+N1 базовых операций с точностью до константы. ВСЕГДА думайте, прежде чем менять иммутабельные объекты, возможно, следует использовать другой тип данных.
    - `+` - конкатенация (сложение) строк.  "Вася"+" Пупкин" = "Вася Пупкин".
    - String.Concat (a1,a2,...,an) - более эффективная конкатенация при условии использования множества аргументов.
    - String.Join(str1, list) - конкатенация через коллекции
    - StringBuilder - специальный тип данных для сборки больших строк и множественных конкатенаций. `StringBuilder sb = new StringBuilder(); sb.append("some string");`
    - `$""` - форматируемая строка. Пример: `$"Name: {name}, Age: {age}"`, компилируется в эффективную сборку строк.
    - `string.Create` - позволяет собирать строки через `Span<char>`, что заметно быстрее конкатенаций и позволяет избежать лишних аллокаций ресурсов, тем не менее есть более удобный и простой `StringBuilder`
    - `==` - оператор сравнения. Несмотря на то, что строки являются ссылочным типом, данный оператор переопределён по умолчанию и сравнивает именно значение строк.
    - `string.Equals(s1, s2, StringComparison.Ordinal)` - более продвинутое сравнение. Можно задать игнор на регистр, например.
    - `string.IsNullOrEmpty(s)` и `string.IsNullOrWhiteSpace(s)` - позволяют проверять строки на пустоту.  
  
  Прочие функции по работе со строками будут рассмотрены в блоке со строками.
- `Math`/`MathF` - классы, предоставляющие математические функции. `Math` работает с `double` (`FP64`) (научные и инженерные вычисления), `MathF` работает с `float` (`FP32`) (игры, графика, прочие простые вычисления)
  - Константы
    - `PI` ~3.14 число Пи.
    - `E` ~2.7182 экспонента
    - `Tau` =2*PI ~6.2831 двойное Пи, оно же Тау, иногда нужно.
  - Методы и функции для работы с числами
    - `Round(x)` - простое округление к ближайшему целому
    - `Round(x, digits)` - округление до digits знака после запятой
    - `Round(x, MidpointRounding)` - указание поведения для округления .5
    - `Ceiling(x)` - наименьшее целое больше x
    - `Floor(x)` - наибольшее целое меньше х
    - `Truncate(x)` - отброс дробной части, возвращает целую часть х
    - `Sign(x)` -  возвращает знак числа. `[-1,0,1]`
    - `Abs(x)` - возвращает модуль числа
    - `Min(a,b)` - возвращает минимальное из a и b
    - `Max(a,b)` - возвращает максимальное из a и b
    - `Clamp(value, min, max)` - ограничение диапазона прим.: `Math.Clamp(0.5, 0, 0.2) -> 0.2` Здесь 0.5 больше 0.2, поэтому вернётся максимальное из диапазона возможных значений.
    - `Pow(x,y)` - возведение x в степень y
    - `Sqrt(x)` - квадратный корень из x
    - `Cbrt(x)` - кубический корень числа
    - `Exp(x)` - возмедение экспоненты в степень x
    - `Log(x)` - натуральный логарифм числа x
    - `Log10(x)` - логарифм числа х по основанию 10
    - `Log(x, y)` - логарифм х по основанию y
    - `Sin(x) Cos(x) Tan(x)` - триада триганометрии. Углы в радианах.
    - `Asin(x) Acos(x) Atan(x)` - обратные тригонометрические функции. Углы в радианах.
    - `Atan2(y,x)` - угол в радианах от оси х к точке x,y
    - `Sinh(x) Cosh(x) Tanh(x)` - гиперболические функции
    - `CopySign(x,y)` - возвращает число по модулю х и знаком y
