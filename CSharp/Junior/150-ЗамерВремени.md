# Замеры времени работы алгоритмов

- `Stopwatch` - основной и базовый инструмент для замера времени выполнения
  
    Пространство имён `System.Diagnostics` предосталвяет класс `Stopwatch` для точного замера времени. В отличие от `DateTime`, он оптимизирован для точности и использует высокоточный аппаратный таймер, если он доступен в системе.

    Пример использования:

    ```csharp
    using System;
    using System.Diagnostics;

    Stopwatch stopwatch = new Stopwatch();
    stopwatch.Start();

    // Код, время выполнения которого нужно измерить

    stopwatch.Stop();

    // Получение результата
    TimeSpan elapsed = stopwatch.Elapsed;
    Console.WriteLine($"Время выполнения: {elapsed}");
    Console.WriteLine($"Миллисекунды: {stopwatch.ElapsedMilliseconds}");
    Console.WriteLine($"Тики: {stopwatch.ElapsedTicks}");

    // более короткий вариант:
    stopwatch = Stopwatch.StartNew();
    // что-то замеряем
    stopwatch.Stop();
    // Пересоздавать не обязательно, таймер можно сбросить:
    stopwatch.Restart(); // Остановка, сброс, перезапуск
    stopwatch.Reset(); // Остановка, сброс, не запускает заново.

    // Если требуется замерить время без аллокации объекта Stopwatch, можно воспользоваться статическими методами

    long start = Stopwatch.GetTimeStamp();
    // чотаделаем
    long end = StopWatch.GetTimeStamp();
    elapsed = Stopwatch.GetElapsedTime(start, end);
    ```

- Высокоточный таймер и частота замеров

    Stopwatch автоматически определяет наличие высокоточного таймера и проверить это можно следующим образом:

    ```csharp
      if (Stopwatch.IsHighResolution) // если true - в системе есть и доступен высокоточный таймер
      {
          Console.WriteLine("Используется высокоточный таймер");
          // Stopwatch.Frequency - частота опроса таймера в секунду (количество тиков), на современных машинах это 10млн опросов с секунду, что составляет 100нс на тик исполнения
          Console.WriteLine($"Частота: {Stopwatch.Frequency} тиков в секунду");
          
          long nanosecPerTick = 1_000_000_000 / Stopwatch.Frequency;
          Console.WriteLine($"Точность: {nanosecPerTick} нс");
      }
      else
      {
          Console.WriteLine("Используется системный таймер (низкое разрешение)");
      }
    ```

    Не следует использовать DateTime.Ticks для замера времени выполнения т.к. используются простые системные часы с разрешением 1-16мс, а так же эта структура хранит количество тиков с 1 января 0001 года, что может дать некачественный результат, когда два опроса DateTime.Ticks вернут одинаковый результат (попадение в 16мс интервал системных часов).

- `BenchmarkDotNet` - профессиональный бенчмаркинг

    Установка:

    ```cmd
    dotnet add package BenchmarkDotNet
    ```

    Минимальный пример:

    ```csharp
      using System;
      using System.Security.Cryptography;
      using BenchmarkDotNet.Attributes;
      using BenchmarkDotNet.Running;

      namespace MyBenchmarks
      {
          public class Md5VsSha256
          {
              private const int N = 10000;
              private readonly byte[] data;
              private readonly SHA256 sha256 = SHA256.Create();
              private readonly MD5 md5 = MD5.Create();

              public Md5VsSha256()
              {
                  data = new byte[N];
                  new Random(42).NextBytes(data);
              }

              [Benchmark]
              public byte[] Sha256() => sha256.ComputeHash(data);

              [Benchmark]
              public byte[] Md5() => md5.ComputeHash(data);
          }

          public class Program
          {
              public static void Main(string[] args)
              {
                  var summary = BenchmarkRunner.Run<Md5VsSha256>();
              }
          }
      }
    ```

    `BenchmarkDotNet` - open-source решение для автоматического замера скорости исполнения написанного кода. Код, время исполнения которого измеряется, запускается несколько раз, результаты усредняются (Mean), выводится ошибка измерения (Error) и стандартное отклонение (StdDev).

    Ключевые элементы:

  - `[Benchmark]` - атрибут перед методами, скорость исполнения которых необходимо измерить
  - `BenchmarkRunner.Run<T>()` - запускает все методы, помеченные для бенчмаркинга и выводит результат в консоль.
  - Запускать бенчмарк необходимо в Release сборке, иначе BenchmarkDotNet выкинет ошибку.

    Пример вывода результатов:

    ```
    | Method | Mean     | Error    | StdDev   |
    |--------|----------|----------|----------|
    | Sha256 | 51.57 us | 0.311 us | 0.291 us |
    | Md5    | 21.91 us | 0.138 us | 0.129 us |
    ```
