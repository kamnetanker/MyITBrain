# Базовая работа с потоками ввода-вывода

`Поток` (`Stream`) ввода-вывода это `абстракция` над некоторой `последовательностью байт`, это `посредник` между `программой` и `внешним миром`(файл, сеть, память), данная `абстракция` определяется как `однонаправленный канал` для передачи `последовательности байт` между `источником` и `приёмником` данных.  

Программы могут читать данные из потока, если они там есть или писать данные в поток, если он есть. Эта абстракция скрывает от программиста все тонкости работы с конкретным устройством, системными вызовами и прочим. Работу по организации взаимодействия с устройствами на себя берёт `операционная система`. Она инициализирует устройства и предоставляет `интерфейс` для взаимодействия - `потоки`.  

`Поток`, обычно, направлен в одну сторону, т.е. позволяет либо `читать` что-то из `потока`, тогда это `поток` ввода, или `писать` что-то в `поток`, тогда это `поток` вывода.  

`Потоки` имеют `буферы` для оптимизации операций, на текущую позицию в последовательности указывает `курсор`, его методы позволяют перемещаться в последовательности байт потока.  

`Поток` - `управляемый` ресурс, требующий явного освобождения через `Dispose()` или `Close()`

Общий родитель всех потоков это класс `System.IO.Stream`. Его нельзя создать, т.к. он абстрактный, но от него наследуются все потоки. Реализации различны - интерфейс взаимодействия единый.

- Пространства имён для работы с потоками

```csharp
using System.IO;              // FileStream, StreamReader, StreamWriter, File, Directory и др.
using System.Text;            // Encoding (UTF8, ASCII, Unicode и др.)
using System.Net.Sockets;     // TcpClient, TcpListener, UdpClient, NetworkStream
using System.Net;             // IPAddress, IPEndPoint, Dns
```

- Управление потоками

    System.IO.Stream имеет следующий `контракт`:

    ```csharp
    public abstract class Stream
    {
        // Чтение
        public abstract int Read(byte[] buffer, int offset, int count);
        // buffer - Буфер, в который при чтении поток запишет данные и при записи откуда прочитает данные
        // offset - Смещение в указанном буфере откуда (или куда) начнётся чтение (или запись)
        // count - Число байт, которые планируется прочитать или записать
        public virtual Task<int> ReadAsync(byte[] buffer, int offset, int count, CancellationToken cancellationToken);

        // Запись
        public abstract void Write(byte[] buffer, int offset, int count);
        public virtual Task WriteAsync(byte[] buffer, int offset, int count, CancellationToken cancellationToken);
        // cancellationToken - Объект, позволяющий отправить в асинхронную операцию сигнал отмены через метод Cancel(). Не все потоки его поддерживают, обычно происходит выброс исключения (OperationCanceledException), а так же потоки, реализация которых устарела, могут игнорировать данный объект и его сигналы.

        // Поиск (перемещение курсора)
        public abstract long Seek(long offset, SeekOrigin origin);
        // origin - SeekOrigin.Begin (начало потока, смещение с начала), SeekOrigin.Current (Текущая позиция курсора, смещение отсчитывается от него), SeekOrigin.End (конец потока, offset <0 чтобы перемещаться от конца к началу)

        // Сброс буфера
        public abstract void Flush();
        public virtual Task FlushAsync(CancellationToken cancellationToken);

        // Закрытие / освобождение
        public virtual void Close();
        public void Dispose();

        // Свойства
        public abstract long Position { get; set; }       // текущая позиция курсора
        public abstract long Length { get; }              // длина потока (байт)
        public abstract bool CanRead { get; }             // поддерживает ли чтение
        public abstract bool CanWrite { get; }            // поддерживает ли запись
        public abstract bool CanSeek { get; }             // поддерживает ли поиск (перемещение)
    }
    ```

    Поток - управляемый ресурс, поэтому его необходимо освобождать, иначе:

  - Утечки памяти
  - Взаимные блокировки
  - Краш приложения

    ```csharp
    // 1 вариант управления потоком
    using (FileStream fs = new FileStream("path\\to\\file.txt", FileMode.Open))
    {
      //Тут чота делаем
    }// Здесь автоматически вызовется fs.Dispose() который закроет поток и освободит ресурсы.

    // 2 Вариант

    FileStream fs2 = null;
    try
    {
      fs2 = new FileStream("path\\to\\file.txt", FileMode.Open);
      // чота делаем
    }
    finally
    {
      fs2?.Dispose(); // через оператор ? происходит проверка - а не нуль ли объект, если нет, то Dispose();
    }
    ```

    `using` - гарантирует вызов `Dispose()`, даже если в процессе работы с потоком будет выброшено исключение.

- Файловые потоки

     `FileStream` - основной класс для работы с файлами. Он предоставляет низкоуровневый доступ к файлам на уровне байт данных.

     ```csharp
     FileStream fs = new FileStream(
        string path,          // Путь к файлу
        FileMode mode,        // В каком режиме открывается файл
        FileAccess access,    // Права доступа к файлу
        FileShare share,      // Доступ к файлу другими процессами
        int bufferSize,       // Размер буфера (по умолчанию 4096 байт, лучше всего использовать кратные 4 KB)
        FileOptions options   // Доп опции для файла      
     );
     ```

     `FileMode`:

  - `FileMode.CreateNew` - Новый файл. Если существует - исключение.
  - `.Create` - Новый файл. Если существует - перезапись.
  - `.Open` - Открыть файл. Если не существует - исключение.
  - `.OpenOrCreate` - Открыть файл. Если не существует - создать.
  - `.Truncate` - Открыть файл и обрезать до 0 длины.
  - `.Append` - Открыть файл и записывать в конец.

    `FileAccess`:

  - `FileAccess.Read` - Только чтение
  - `.Write` - Только запись
  - `.ReadWrite` - Чтение и запись.

    `FileShare`:

  - `FileShare.None` - Другие процессы не имеют доступа
  - `.Read` - Другие процессы могут только читать
  - `.Write` - Другие процессы могут только писать
  - `.ReadWrite` - Другие процессы могут читать и писать
  - `.Delete` - Другие процессы могут удалять файл.

    `FileOptions`:

  - `FileOptions.None` - Без дополнительных опций
  - `.Asynchronous` - асинхронный ввод-вывод
  - `.WriteThrough` - Запись сразу на диск, без кэша ОС
  - `.SequentialScan` - применить оптимизации для последовательного доступа
  - `.RandomAccess` - применить оптимизации для произвольного доступа

    Пример работы с файловыми потоками:

    ```csharp
    using System.IO;
    using System.Text;

    string filePath = "data.bin";

    // === ЗАПИСЬ БАЙТОВ ===
    byte[] dataToWrite = { 0x48, 0x65, 0x6C, 0x6C, 0x6F }; // "Hello"

    using (FileStream fs = new FileStream(filePath, FileMode.Create, FileAccess.Write))
    {
        fs.Write(dataToWrite, 0, dataToWrite.Length);
    }

    // === ЧТЕНИЕ БАЙТОВ ===
    using (FileStream fs = new FileStream(filePath, FileMode.Open, FileAccess.Read))
    {
        byte[] buffer = new byte[fs.Length];
        int bytesRead = fs.Read(buffer, 0, buffer.Length);
        string result = Encoding.UTF8.GetString(buffer, 0, bytesRead);
        Console.WriteLine(result); // "Hello"
    }

    // === ОПТИМАЛЬНОЕ ЧТЕНИЕ С БУФЕРОМ (для больших файлов) ===
    using (FileStream fs = new FileStream(filePath, FileMode.Open, FileAccess.Read))
    {
        byte[] buffer = new byte[4096]; // читаем по 4KB за раз
        int bytesRead;
        while ((bytesRead = fs.Read(buffer, 0, buffer.Length)) > 0)
        {
            // обрабатываем bytesRead байт из buffer
            Console.WriteLine($"Прочитано {bytesRead} байт");
        }
    }
    ```

- Работа с текстом (Текстовый ввод-вывод)

  `StreamReader` и `StreamWriter` - адаптеры над `FileStream`, которые автоматически преобразуют последовательность байт в символы с учётом кодировки

  ```csharp
  using System.IO;
  using System.Text;

  string filePath = "text.txt";

  using (StreamWriter writer = new StreamWriter(filePath, false, Encoding.UTF8))
  {
      writer.WriteLine("Первая строка");
      writer.WriteLine("Вторая строка");
      writer.Write("Третья строка без перевода строки");
  }
  // Параметры: путь, append (добавлять или перезаписать), кодировка

  using (StreamReader reader = new StreamReader(filePath, Encoding.UTF8))
  {
      // Чтение всего файла
      string allText = reader.ReadToEnd();
      
      // Или построчно
      string? line;
      while ((line = reader.ReadLine()) is not null)
      {
          Console.WriteLine(line);
      }
  }
  ```

  Для работы с маленькими файлами есть упрощённый статический класс `File`, позволяющий взаимодействовать с файлами однострочными командами:

  ```csharp
  using System.IO;
  using System.Text;

  string path = "file.txt";

  // Запись текста
  File.WriteAllText(path, "Hello World", Encoding.UTF8);
  File.WriteAllLines(path, new[] { "Line 1", "Line 2" });

  // Чтение текста
  string content = File.ReadAllText(path, Encoding.UTF8);
  string[] lines = File.ReadAllLines(path);

  // Запись байтов
  File.WriteAllBytes(path, new byte[] { 0x48, 0x65 });

  // Чтение байтов
  byte[] data = File.ReadAllBytes(path);
  ```

  Важно: Эти команды загружают весь файл в память, поэтому для работы с файлами больше 10-20 МБ следует использовать `FileStream`.

- Сетевые потоки

  - TCP - сетевой протокол с установлением соединения. Гарантирует доставку и порядок пакетов данных. Ниже пример клиент-серверного взаимодействия по этому протоколу. Приведён пример серверного кода и клиентского.

      `TcpListener` и `TcpClient` - как это работает?

      Сервер создаёт объект `TcpListener` на `IPEndPoint` (точка подключения), который задаётся  `Ip-адресом` и `портом`, на котором сервер будет `слушать` входящие подключения. Если `Ip-адрес` не указан, то сервер будет слушать все сетевые интерфейсы на указанном порту.

      `IPEndPoint` - структура, объединяющая IP-адрес и порт. Используется для указания адресата при отправке данных или для получения адреса отправителя, когда получашь данные.

      Далее варианты создания `TcpListener`:

      ```csharp
      // Способ 1: IP-адрес и порт (рекомендуется)
      IPAddress localAddr = IPAddress.Parse("127.0.0.1");
      int port = 8080;
      TcpListener listener = new TcpListener(localAddr, port);

      // Способ 2: IPEndPoint
      IPEndPoint localEndPoint = new IPEndPoint(IPAddress.Any, 8080);
      // IPAddress - указание на каком сетевом интерфейсе слушать, IPAddress.Any - на любом, .Loopback - на локальном (127.0.0.1)
      // port - номер порта (16-ти битное целое от 1 до 65535). Порты ниже 1024 требуют админских прав для работы.
      TcpListener listener2 = new TcpListener(localEndPoint);

      // Способ 3: Только порт (слушаем на всех IP-адресах)
      TcpListener listener3 = new TcpListener(port);
      ```

      Далее варианты создания `TcpClient`:

      ```csharp
      // Способ 1: Создаём и сразу подключаемся (не рекомендуется)
      TcpClient client = new TcpClient("127.0.0.1", 8080);

      // Способ 2: Создаём без подключения, затем подключаемся (рекомендуется)
      TcpClient client = new TcpClient();
      await client.ConnectAsync("127.0.0.1", 8080);
      ```

      Параметры можно задать следующие:
    - `hostname` - имя хоста или ip-адрес в виде строки
    - `IPAddress` - можно как объект создать и передать.
    - `port` - порт, на котором будет сервер с которым мы устанавливаем соединение
    - для асинхронных методов `cancellationToken` - для отмены подключений.

      После того, как сервер создал `TcpListener` - он должен начать "слушать" подключения. Когда происходит подключение - создаётся объект `TcpClient`, который точно определяет клиента, с которым установлено соединение и пакеты данных этого клиента будут обрабатываться через этот объект. Ниже пример того, как принимаются подключения и объект-клиент передаётся на обработку в сторонний поток (это не обязательно, клиентов можно обрабатывать последовательно, но пример с параллельностью имеет смысл показать):

      ```csharp
      while (true)
      {
          // Ожидаем подключения нового клиента
          TcpClient client = await listener.AcceptTcpClientAsync();

          // У каждого клиента свой TcpClient. Теперь мы знаем, кто это!
          // Передаём этого клиента в отдельную задачу для обработки
          _ = Task.Run(() => HandleClient(client));
          // Task.Run(...) возвращает объект, позволяющий управлять созданным потоком, но так как он нам не нужен, то сбрасывается через _
      }
      ```

      Далее, когда соединение установлено, клиент и сервер получают поток ввода-вывода, через который обмениваются данными. `NetworkStream` - собственно объект-канал для передачи данных через `TcpClient`. Он не поддерживает произвольный доступ к потоку (флаг CanSeek == false) и не владеет сокетом, через который происходит обмен данными.

      Серверный пример:

      ```csharp
      // Сервер
      NetworkStream stream = client.GetStream();

      // Чтение данных
      byte[] buffer = new byte[1024];
      int bytesRead = await stream.ReadAsync(buffer, 0, buffer.Length, cancellationToken);
      
      // Отправка данных
      byte[] responseData = Encoding.UTF8.GetBytes("Hello, client!");
      await stream.WriteAsync(responseData, 0, responseData.Length, cancellationToken);
      ```

      Клиентский пример:

      ```csharp
      // Клиент
      NetworkStream stream = client.GetStream();

      string message = "Hello, TCP!";
      byte[] data = Encoding.UTF8.GetBytes(message);
      await stream.WriteAsync(data, 0, data.Length);

      // Чтение ответа
      byte[] buffer = new byte[1024];
      int bytesRead = await stream.ReadAsync(buffer, 0, buffer.Length);
      string response = Encoding.UTF8.GetString(buffer, 0, bytesRead);
      Console.WriteLine($"Ответ сервера: {response}");
      ```

      Пример клиент-серверного взаимодействия:

      ```csharp
      // Сервер (приём данных)
      using System.Net;
      using System.Net.Sockets;
      using System.Text;

      int port = 8080;
      TcpListener listener = new TcpListener(IPAddress.Any, port);
      listener.Start();
      Console.WriteLine($"TCP сервер запущен на порту {port}");

      while (true)
      {
          TcpClient client = await listener.AcceptTcpClientAsync();
          Console.WriteLine("Клиент подключился");

          NetworkStream stream = client.GetStream();
          byte[] buffer = new byte[1024];
          int bytesRead = await stream.ReadAsync(buffer, 0, buffer.Length);
          
          string received = Encoding.UTF8.GetString(buffer, 0, bytesRead);
          Console.WriteLine($"Получено: {received}");

          // Ответ клиенту
          string response = "OK";
          byte[] responseData = Encoding.UTF8.GetBytes(response);
          await stream.WriteAsync(responseData, 0, responseData.Length);

          stream.Close();
          client.Close();
      }
      ```

      Клиент:

      ```csharp
      using System.Net.Sockets;
      using System.Text;

      string serverIP = "127.0.0.1";
      int port = 8080;

      TcpClient client = new TcpClient();
      await client.ConnectAsync(serverIP, port);

      NetworkStream stream = client.GetStream();

      string message = "Hello, TCP!";
      byte[] data = Encoding.UTF8.GetBytes(message);
      await stream.WriteAsync(data, 0, data.Length);

      // Чтение ответа
      byte[] buffer = new byte[1024];
      int bytesRead = await stream.ReadAsync(buffer, 0, buffer.Length);
      string response = Encoding.UTF8.GetString(buffer, 0, bytesRead);
      Console.WriteLine($"Ответ сервера: {response}");

      stream.Close();
      client.Close();
      ```

      Так же ниже приведён комплексный пример по обработке множества клиентов в фоне:

      ```csharp
      // Запускаем бесконечный цикл приёма подключений
      while (true)
      {
          TcpClient client = await listener.AcceptTcpClientAsync();
          Console.WriteLine($"Клиент {client.Client.RemoteEndPoint} подключился.");

          // Запускаем обработку клиента в фоновой задаче
          _ = Task.Run(() => HandleClientAsync(client));
      }

      async Task HandleClientAsync(TcpClient client)
      {
          // Получаем уникальный идентификатор клиента
          var clientId = client.Client.RemoteEndPoint?.ToString() ?? "Unknown";
          Console.WriteLine($"[{clientId}] Начата обработка.");

          try
          {
              using (client)
              using (NetworkStream stream = client.GetStream())
              {
                  // Цикл обмена данными с этим конкретным клиентом
                  byte[] buffer = new byte[1024];
                  int bytesRead;
                  while ((bytesRead = await stream.ReadAsync(buffer, 0, buffer.Length)) > 0)
                  {
                      string receivedMessage = Encoding.UTF8.GetString(buffer, 0, bytesRead);
                      Console.WriteLine($"[{clientId}] Получено: {receivedMessage}");
                      // ... обработка ...
                  }
              }
          }
          catch (Exception ex)
          {
              Console.WriteLine($"[{clientId}] Ошибка: {ex.Message}");
          }
          finally
          {
              Console.WriteLine($"[{clientId}] Соединение закрыто.");
          }
      }
      ```

  - UDP - сетевой протокол без установления соединения. НЕ гарантирует доставку и порядок пакетов.

      Каждый пакет(датаграмма) содержит адрес отправителя из-за того, что соединение не устанавливается.

      Пример создания `UdpClient`:

      ```csharp
      // Привязываемся к локальному порту 11000 (для сервера)
      UdpClient server = new UdpClient(11000);

      // Клиент без привязки к порту (система назначит автоматически)
      UdpClient client = new UdpClient();

      // Так же можно передать IPEndPoint с Ip и портом
      ```

      Далее клиент просто отправляет данные.

      ```csharp
      byte[] data = Encoding.UTF8.GetBytes("Hello, UDP!");
      await client.SendAsync(data, data.Length, "127.0.0.1", 11000); // Вместо строки с адресом и портом, можно передать IPEndPoint
      ```

      Сервер же, в свою очередь, просто ожидает, когда к нему прилетят данные:

      ```csharp
      // Асинхронный приём (рекомендуется)
      UdpReceiveResult result = await server.ReceiveAsync();
      byte[] receivedData = result.Buffer;
      IPEndPoint senderEndPoint = result.RemoteEndPoint;

      Console.WriteLine($"Получено от {senderEndPoint.Address}:{senderEndPoint.Port}");
      ```

      Сервер

      ```csharp
      using System.Net;
      using System.Net.Sockets;
      using System.Text;

      UdpClient server = new UdpClient(11000);
      IPEndPoint remoteEndPoint = new IPEndPoint(IPAddress.Any, 0);

      Console.WriteLine("UDP сервер ожидает данные...");

      byte[] receiveBytes = server.Receive(ref remoteEndPoint);
      string receivedData = Encoding.UTF8.GetString(receiveBytes);
      Console.WriteLine($"Получено от {remoteEndPoint}: {receivedData}");

      // Отправка ответа
      string response = "OK";
      byte[] responseData = Encoding.UTF8.GetBytes(response);
      server.Send(responseData, responseData.Length, remoteEndPoint);

      server.Close();
      ```

      Клиент

      ```csharp
      using System.Net;
      using System.Net.Sockets;
      using System.Text;

      string serverIP = "127.0.0.1";
      int serverPort = 11000;

      UdpClient client = new UdpClient();
      string message = "Hello, UDP!";
      byte[] data = Encoding.UTF8.GetBytes(message);

      await client.SendAsync(data, data.Length, serverIP, serverPort);

      // Приём ответа
      IPEndPoint remoteEndPoint = new IPEndPoint(IPAddress.Any, 0);
      byte[] receiveBytes = client.Receive(ref remoteEndPoint);
      string response = Encoding.UTF8.GetString(receiveBytes);
      Console.WriteLine($"Ответ сервера: {response}");

      client.Close();
      ```

- Сетевое программирование база на примере пересылки изображения по сети с использованием TCP
  
  Сервер (принимает изображение)

  ```csharp
  using System.Net;
  using System.Net.Sockets;
  using System.Text;

  int port = 8080;
  string saveDir = @"C:\Temp\ReceivedImages";
  Directory.CreateDirectory(saveDir);

  TcpListener listener = new TcpListener(IPAddress.Any, port);
  listener.Start();
  Console.WriteLine("Сервер запущен. Ожидание файлов...");

  while (true)
  {
      TcpClient client = await listener.AcceptTcpClientAsync();
      Console.WriteLine("Клиент подключился");

      NetworkStream stream = client.GetStream();
      
      // 1. Читаем размер файла (8 байт)
      byte[] sizeBuffer = new byte[8];
      await stream.ReadAsync(sizeBuffer, 0, 8);
      long fileSize = BitConverter.ToInt64(sizeBuffer, 0);
      Console.WriteLine($"Размер файла: {fileSize} байт");

      // 2. Читаем само изображение
      byte[] imageBuffer = new byte[fileSize];
      int bytesRead = 0;
      while (bytesRead < fileSize)
      {
          int read = await stream.ReadAsync(imageBuffer, bytesRead, (int)(fileSize - bytesRead));
          if (read == 0) break;
          bytesRead += read;
      }

      // 3. Сохраняем
      string fileName = $"image_{DateTime.Now:yyyyMMddHHmmss}.jpg";
      string fullPath = Path.Combine(saveDir, fileName);
      await File.WriteAllBytesAsync(fullPath, imageBuffer);
      Console.WriteLine($"Файл сохранён: {fullPath}");

      // 4. Отправляем подтверждение
      byte[] ack = Encoding.UTF8.GetBytes("OK");
      await stream.WriteAsync(ack, 0, ack.Length);

      stream.Close();
      client.Close();
      Console.WriteLine("Соединение закрыто.");
  }
  ```

  ```csharp
  using System.Net;
  using System.Net.Sockets;
  using System.Text;

  string serverIP = "127.0.0.1";
  int port = 8080;

  // 1. Загружаем файл в память

  TcpClient client = new TcpClient();

  // 2. Отправляем размер (8 байт)
  byte[] sizeData = BitConverter.GetBytes(imageData.Length);
  await stream.WriteAsync(sizeData, 0, sizeData.Length);

  // 3. Отправляем изображение
  await stream.WriteAsync(imageData, 0, imageData.Length);
  Console.WriteLine("Изображение отправлено.");

  // 4. Читаем подтверждение
  byte[] ackBuffer = new byte[2];
  int bytesRead = await stream.ReadAsync(ackBuffer, 0, ackBuffer.Length);
  string ack = Encoding.UTF8.GetString(ackBuffer, 0, bytesRead);
  Console.WriteLine($"Ответ сервера: {ack}");

  stream.Close();
  client.Close();
  ```
