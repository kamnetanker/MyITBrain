# Рефлексия и атрибуты

`Атрибуты` - механизм добавления `метаданных` (декларативной информации) к коду: к сборке, к типу данных, методам и свойствам. В блоке про сериализацию уже встречались атрибуты, которыми помечались поля классов. 

Свойства атрибутов:

- Атрибуты добавляют метаданные в программу (в том числе в сборку)
- Могут применяться к сборке в целом, модулям и отдельным элементам (классы, поля, методы)
- Могут принимать аргументы
- Позволяет программе осуществлять рефлексию, через метаданные.
- В качестве параметров атрибутов могут выступать следующие типы данных: bool, byte, char, double, float, int, long, short, string, System.Type, object, public enum. Другие не поддерживаются.

Синтаксис: `[attribute(argument=value)]` - в квадратных скобках указывается атрибут, к которому добавляется аргумент и его значение. Аргумент и значение, зачастую, не обязательны и минимальный атрибут имеет вид: `[attribute]`

Пример:

```c#
[Serializable]
public class SampleClass
{
    /*
    Здесь к атрибуту добавлено значение аргумента
    имя аргументов указывать не обязательно, если они передаются в
    сигнатурном порядке (т.е. как в методах)
    */
    [Obsolete("Use NewMethod instead")] 
    public void OldMethod() { }
}
```

По соглашению имена классов атрибутов заканчиваются суффиксом Attribute, но при использовании суффикс можно опускать:

- `[ObsoleteAttribute]` ⇔ `[Obsolete]`
- `[SerializableAttribute]` ⇔ `[Serializable]`

Встроенные C# атрибуты:

- `[Obsolete]` Помечает элемент программы как устаревший. Компилятор выдаст предупреждение или ошибку
- `[Serializable]` Указывает, что тип данных можно сериализовать. (Deprecated)
- `[NonSerialized]` Помечает поле как не участвующее в сериализации (Deprecated)
- `[JsonIgnore]` - современный способ определить поле, как несериализуемое
- `[DllImport]` объявляет внешнюю функцию из неуправляемой dll библиотеки
- `[Conditional]` Помечает метод, как вызываемый только при определённой конфигурации компиляции
- `[AttributeUsage] ` Ограничивает применение пользовательских атрибутов

Атрибуты можно создавать собственноручно:

```c#
[AttributeUsage(AttributeTargets.Class | AttributeTargets.Struct, AllowMultiple = true)]
public class AuthorAttribute : Attribute
{
    private string name;
    public double Version;

    public AuthorAttribute(string name)
    {
        this.name = name;
        Version = 1.0;
    }

    public string GetName() => name;
}

[Author("H. Ackerman")]
public class FirstClass { }

[Author("H. Ackerman"), Author("M. Knott", Version = 2.0)]
public class ThirdClass { }
```

Здесь:

`AttributeTargets` - позиционный параметр, определяющий к каким элементам можно применять атрибут: `Class`, `Method`, `Property`, `Assembly` и др., комбинируются через `|`

`AllowMultiple` - именованный параметр, определяющий можно ли применять атрибут несколько раз к одному элементу.

`Inherited` - именованный, наследуется ли атрибут производными классами.

При использовании суффикс `Attribute` опускается. Позиционные параметры (как `name`) указываются первыми и в фиксированном порядке, именованные — в любом порядке

---

Рефлексия - механизм, позволяющий программе в ходе работы исследовать свою структуру. Основывается этот механизм на Атрибутах, т.е. метаданных исследуемых элементов.

Основные возможности:
- Динамически создавать экземпляры типов
- Получать информацию о сборках, модулях и типах
- Вызывать методы и обращаться к полям/свойствам по имени
- Читать атрибуты, применённые к элементам кода

Ключевое пространство имён: `System.Reflection` содержащее следующее:

- `Type` - класс, предоставляет информацию об объявленном типе (класс это или интерфейс, перечисление и прочее)
- `Assembly` - предоставляет информацию о сборке
- `MethodInfo` - информация о методе
- `PropertyInfo` - информация о свойствах
- `FieldInfo` - информация о поле
- `ConstructorInfo` - информация о конструкторе

```c#
/*
Примеры получения информации о типе, сборке
*/

// Через GetType() (есть у любого объекта)
int i = 42;
Type type = i.GetType();
Console.WriteLine(type); // System.Int32

// Через typeof (на этапе компиляции)
Type stringType = typeof(string);

// Через Type.GetType() (по строковому имени)
Type customType = Type.GetType("MyNamespace.MyClass");

Assembly assembly = typeof(int).Assembly;
Console.WriteLine(assembly.FullName);
// System.Private.CoreLib, Version=7.0.0.0, Culture=neutral, PublicKeyToken=...
```

Атрибуты и рефлексия - два брата акробата, попугайчики неразлучники, первый не имеет смысла без второго, а второй не фунциклирует без первого. Ключевой метод для взаимодействия: `Attribute.GetCustomAttributes()` - возвращает массив атрибутов, примененных к элементу.

```c#
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Struct, AllowMultiple = true)]
public class AuthorAttribute : Attribute
{
    private string name;
    public double Version;

    public AuthorAttribute(string name)
    {
        this.name = name;
        Version = 1.0;
    }

    public string GetName() => name;
}

[Author("H. Ackerman")]
public class FirstClass { }

[Author("H. Ackerman"), Author("M. Knott", Version = 2.0)]
public class ThirdClass { }

class Program
{
    static void Main()
    {
        PrintAuthorInfo(typeof(FirstClass));
        PrintAuthorInfo(typeof(ThirdClass));
    }

    static void PrintAuthorInfo(Type t)
    {
        Console.WriteLine($"Author information for {t}");
        
        // Читаем все атрибуты, применённые к типу t
        Attribute[] attrs = Attribute.GetCustomAttributes(t);
        
        foreach (Attribute attr in attrs)
        {
            if (attr is AuthorAttribute author)
            {
                Console.WriteLine($"  {author.GetName()}, version {author.Version:F2}");
            }
        }
    }
}
/* Вывод:
Author information for FirstClass
  H. Ackerman, version 1.00
Author information for ThirdClass
  M. Knott, version 2.00
  H. Ackerman, version 1.00
*/
```

Важно: атрибут не создаётся в момент применения — он создаётся в момент вызова GetCustomAttributes через рефлексию. Т.е. до момента обработки атрибутов через рефлексию, тысячи объектов типа атрибут не висят в памяти.

Когда код компилируется в IL (Intermediate Language) не все модификаторы доступа сохраняются в том виде, в котором они описаны и меняют своё название:

- `internal` - IL: `Assembly`, Reflection: `MethodBase.IsAssembly`
- `protected internal` - IL: `FamilyOrAssembly`, Reflection: `MethodBase.IsFamilyOrAssembly`

Это надо помнить и знать.

**Где применяется в реальных проектах**
- Сериализация — атрибуты [Serializable], [NonSerialized], [DataMember]
- ORM (Entity Framework) — атрибуты [Table], [Column], [Key]
- Валидация — атрибуты [Required], [StringLength], [Range]
- Тестирование — атрибуты [Test], [Fact], [Theory]
- Swagger/OpenAPI — атрибуты для документирования API
- Плагины — поиск классов с определённым атрибутом через рефлексию для динамической загрузки
- Dependency Injection — чтение атрибутов для автоматической регистрации сервисов
- AOP (Aspect-Oriented Programming) — атрибуты для логирования, кэширования, проверки прав

**Производительность и предостережения**
- Рефлексия медленнее прямого вызова — используйте с умом, особенно в горячих путях
- Кешируйте результаты рефлексии (объекты Type, MethodInfo и т.д.), если вызываете многократно
- GetCustomAttributes создаёт новые экземпляры атрибутов при каждом вызове
- Для высоконагруженных сценариев рассмотрите Source Generators (генерация кода на этапе компиляции) как альтернативу рефлексии
